## 摘要
在线自蒸馏（OPSD）在模型自身推出数据上训练，并使用冻结副本提供以参考目标为条件的密集词元级目标。该方法在LLM推理中效果良好，但直接扩展到多模态大语言模型（MLLMs）可能会产生捷径：特权目标可能主要基于文本参考目标而非图像引导词元。我们提出ViGOS，一种面向MLLM后训练的视觉引导在线自蒸馏框架。学生模型首先编写视觉描述，然后推理出最终答案。对于有效推出，仅图像的感知教师监督描述，而特权推理教师在同一学生前缀上监督推理和最终答案。仅对无效推出使用参考教师以恢复输出格式。在通用视觉-语言、专家推理、视觉数学、空间定位和视觉-语言先验基准上，ViGOS保持了OPSD的主要优势，并在易产生捷径的场景中改善了基于图像的行为。

## Abstract
On-policy self-distillation (OPSD) trains a model on its own rollouts and uses a frozen copy to provide dense token-level targets conditioned on a reference target. This works well for LLM reasoning, but a direct extension to multimodal large language models (MLLMs) can create a shortcut: the privileged target may guide tokens mainly based on the text reference target rather than the image. We propose ViGOS, a visually grounded OPSD framework for MLLM post-training. The student first writes a visual description and then reasons toward the final answer. For valid rollouts, an image-only perception teacher supervises the description, while a privileged reasoning teacher supervises the reasoning and final answer on the same student prefix. A reference teacher is used only for invalid rollouts to recover the output format. Across general vision-language, expert reasoning, visual math, spatial grounding, and visual-language-prior benchmarks, ViGOS keeps the main benefits of OPSD and improves image-grounded behavior in shortcut-prone settings.

- **背景**：OPSD在LLM推理中很有效，它通过冻结的教师模型在学生的自采样轨迹上提供密集监督，无需外部强教师。但MLLM同时接收图像和文本输入，文本路径（问题、选项、参考答案）通常比图像更容易被模型利用。直接将OPSD用于MLLM可能导致教师跳过图像证据，直接根据参考答案来引导推理和答案。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
将学生生成的响应分为三个结构化的部分：视觉描述（description, \(d\)）、推理过程（reasoning, \(r\)）和最终答案（answer, \(a\)）。训练过程中，三个不同的教师模型分别监督不同的词元段：
- **图像专用感知教师（Image-only Perception Teacher）**：仅以图像为外部上下文，监督描述部分的词元，确保描述真正基于图像。
- **特权推理教师（Privileged Reasoning Teacher）**：以图像、问题和参考答案为上下文，监督推理和答案部分的词元，提供基于答案的引导，但只在描述段之后生效。
- **参考教师（Reference Teacher）**：仅用于格式无效的rollout（如缺失分隔符），提供逆向KL恢复信号，防止格式漂移。

### 关键技术细节
1. **结构化推出格式**：学生输出必须包含 `<description>...</description>`, `<think>...</think>`, 和 `\boxed{}` 三个部分。
2. **词元段解析**：训练时，通过解析分隔符得到描述、推理、答案的词元索引集合 \(T_d, T_r, T_a\)，以及无效标志 \(m_{\text{inv}}(y)\)。
3. **三种教师上下文**：
   - \(c_{\text{img}} = I\) （仅图像）
   - \(c_{\text{rea}} = (I, x, a^\star, \pi_{\text{rea}})\)
   - \(c_{\text{ref}} = (I, x, a^\star, \pi_{\text{out}})\)
4. **损失函数**：
   - 感知损失（有效rollout时，作用于 \(T_d\)）：\(\mathcal{L}_{\text{perc}} = \mathbb{E}\left[(1-m_{\text{inv}})\sum_{t\in T_d} D_{KL}(q_{\text{img},t} \| p_{\theta,t})\right]\)
   - 推理损失（有效rollout时，作用于 \(T_r \cup T_a\)）：\(\mathcal{L}_{\text{rea}} = \mathbb{E}\left[(1-m_{\text{inv}})\sum_{t\in T_r\cup T_a} D_{KL}(q_{\text{rea},t} \| p_{\theta,t})\right]\)
   - 参考回退损失（无效rollout时，作用于所有词元）：\(\mathcal{L}_{\text{ref}} = \mathbb{E}\left[m_{\text{inv}}\sum_{t\in T_y} D_{KL}(p_{\theta,t} \| q_{\text{ref},t})\right]\)
   - 总损失：\(\mathcal{L}_{\text{ViGOS}} = \lambda_{\text{perc}}\mathcal{L}_{\text{perc}} + \lambda_{\text{rea}}\mathcal{L}_{\text{rea}} + \lambda_{\text{ref}}\mathcal{L}_{\text{ref}}\)
5. **特权答案泄漏率（PALR）**：提出该诊断指标，通过反事实对比（错误答案、错误图像）来量化答案驱动监督在总监督中的比例，用于分析捷径风险。ViGOS相比OPSD显著降低了PALR。

### 算法流程（文字说明）
1. 从训练数据中采样图像-问题-答案三元组。
2. 学生模型根据结构化格式自行采样一个回复 \(y = (d, r, a)\)。
3. 解析器提取词元段掩码，判断格式有效性。
4. 如果有效，分别使用图像专用感知教师计算描述段KL散度，使用特权推理教师计算推理+答案段KL散度，求和得到ViGOS损失。
5. 如果无效，使用参考教师计算整个回复的逆向KL散度作为损失。
6. 梯度更新学生模型参数（教师模型冻结）。
7. 推理时仅使用学生模型，按相同结构化格式生成回复。

## 6. 论文的主要结论与发现

1. **识别了多模态OPSD的捷径风险**：通过PALR诊断，发现标准OPSD中17%~26%的密集校正由特权答案驱动，可能弱化图像使用。
2. **ViGOS有效缓解捷径**：在保持OPSD主体基准性能的同时，显著降低了PALR（3B: 17.26%→6.33%; 7B: 26.01%→7.56%），并在ViLP等先验冲突任务中取得更高Score，且不降低Prior。

4. **ViGOS在通用基准上保持甚至超越OPSD**：在8个主基准上，ViGOS的Pass@5和Avg@5与OPSD持平或略优，尤其在需要图像理解的任务（如RealWorldQA、MMSI、CV-Bench）上提升明显。
5. **结构化提示本身不足以解决问题**：同提示控制实验显示，仅改变提示格式而不改变监督方式（OPSD使用结构化提示）仍会在先验冲突场景中表现不佳，说明解耦监督是核心。

## 8. 不足与局限：包括实验覆盖、偏差风险、应用限制等

- **主干覆盖有限**：仅使用Qwen2.5-VL系列（3B、7B），未在其他主流MLLM（如LLaVA、InternVL等）上验证通用性。
- **训练数据单一**：仅使用Vision-SR1-47K，未尝试不同来源和规模的数据集。
- **计算开销**：训练时需运行三个教师模型的前向传播，计算量约是标准OPSD的3倍（每个词元需三个教师分布），尽管只针对有效rollout的部分词元，但整体开销仍较大。
- **对描述质量的依赖性**：如果学生模型生成的初始描述不正确或不完整，后续推理可能基于错误信息。图像专用教师仅提供分布匹配，无法纠正描述事实错误。
- **PALR诊断的局限性**：仅测量答案驱动 vs 图像驱动校正，未考虑其他潜在捷径（如语言先验布局、数据集分布偏差）。反事实选择（单张错位图像）可能不鲁棒。
- **基准覆盖缺失**：未评估如图片生成、视频理解、多轮对话等更原始的多模态任务，也未在人类评估或实际产品环境中测试。
- **平衡问题**：在部分任务（如MMVet 3B）上Pass@5略低于OPSD，说明ViGOS可能在某些场景下牺牲了答案搜索广度以换取稳定性。
- **应用限制**：结构化输出格式要求可能不适用于所有下游任务（如自由文本生成）；且需要设计合适的输出解析器，对格式错误敏感。
