## 摘要
经验驱动的自我进化对于大型语言模型（LLM）智能体通过开放世界交互进行改进至关重要。然而，现有的经验学习方法大多依赖于单智能体循环，即同一智能体执行任务、总结结果并确定记忆内容。这种设置使得智能体容易陷入“自我确认陷阱”：错误但自洽的轨迹被误认为是成功经验，导致在检索和重用过程中累积错误。为解决这一问题，我们提出了EDV，一种用于可靠经验学习的执行-蒸馏-验证框架。在执行阶段，多个异构智能体并行探索同一任务空间，生成多样化的候选轨迹。在蒸馏阶段，一个专门的第三方智能体对这些轨迹进行比较分析，生成候选经验，从而减少以执行者为中心的总结偏差。在验证阶段，执行组通过共识机制验证候选经验，只有通过验证的经验才会被写入共享或私有记忆。通过解耦三个阶段，EDV将经验学习从孤立的自我反思转变为协作构建，在记忆插入前过滤错误和噪声内容。我们在三个具有挑战性的长时程基准测试上评估了EDV：tau2-bench、Mind2Web和MMTB。结果表明，EDV持续优于强基线，验证了可靠经验构建对于鲁棒的智能体自我进化至关重要。我们的代码可在https://github.com/shidingz/EDV获取。

## Abstract
Experience-driven self-evolution is critical for large language model (LLM) agents to improve through open-world interaction. However, existing experience learning methods mostly rely on single-agent loops, where the same agent executes tasks, summarizes outcomes, and determines memory content. This setup makes agents vulnerable to the Self-Confirmation Trap: wrong-but-self-consistent trajectories are misidentified as successful experience, leading to cumulative errors during retrieval and reuse. To address this issue, we propose EDV, an Execute-Distill-Verify framework for reliable experience learning. In the Execute stage, multiple heterogeneous agents explore the same task space in parallel to generate diverse candidate trajectories. In the Distill stage, a dedicated third-party agent comparatively analyzes these trajectories to produce candidate experiences, reducing executor-centric summarization bias. In the Verify stage, the execution group validates candidates via a consensus mechanism, and only approved experiences are written into shared or private memory. By decoupling the three stages, EDV transforms experience learning from isolated self-reflection into collaborative construction, filtering erroneous and noisy content before memory insertion. We evaluate EDV on three challenging long-horizon benchmarks: tau2-bench, Mind2Web and MMTB. Results show EDV consistently outperforms strong baselines, validating that reliable experience construction is essential for robust agent self-evolution. Our code is available at https://github.com/shidingz/EDV.

-   **核心问题**：论文揭示了基于大语言模型（LLM）的智能体在自我进化过程中普遍存在的一个关键缺陷——**“自我确认陷阱”**。
-   **背景与动机**：
    -   现有单智能体经验学习范式（如ReasoningBank）通常采用“执行-总结-记忆-重用”的闭环。在这些系统中，同一个智能体既要负责执行任务，又要负责总结和评价执行结果，并决定哪些经验值得写入长期记忆。
    -   在缺乏明确外部反馈信号的开放世界环境中，这种高度耦合的机制是脆弱的。智能体倾向于将**内部一致但实际上错误的轨迹**（`wrong-but-self-consistent trajectories`）错误地判断为成功经验。
    -   这种错误经验一旦进入长时记忆，就会在后续任务中被反复检索和重用，导致错误不断累积和放大。论文将这种系统性的失效模式正式定义为“自我确认陷阱”。
    -   因此，论文认为经验学习的关键挑战不仅仅是“收集更多经验”，而是“更可靠地构建经验”，在经验写入记忆之前压制错误和噪声。

#### 2. 论文提出的方法论

-   **核心思想**：提出 **EDV (Execute-Distill-Verify)** 框架，其核心思想是将单智能体的自我反思循环，解耦为一个**协作式经验构建**过程。通过引入多智能体协作和角色分工，对记忆库的写入设立更高的门槛，从根本上抑制自我确认陷阱。

-   **关键技术细节（三个解耦的阶段）**：
    1.  **Execute (执行阶段)**：
        -   **目的**：生成多样化的候选轨迹，扩大解空间覆盖率。
        -   **方法**：从异构智能体池（不同基础模型或提示策略）中随机采样一个子集，形成“执行组”。每个智能体独立地在同一任务空间中交互，生成各自的完整轨迹。异构性保证了不同成功和失败模式的暴露。
    2.  **Distill (蒸馏阶段)**：
        -   **目的**：从多条轨迹中对比分析，提炼出候选经验，减少单一视角的偏差。
        -   **方法**：从智能体池中随机选取一个与执行组不同的第三方智能体，作为“蒸馏智能体”。该智能体不简单地总结单条最佳轨迹，而是对执行组产生的**多条轨迹进行对比分析**，识别出有价值的差异、成功模式和失败教训，输出一组“候选经验”。
    3.  **Verify (验证阶段)**：
        -   **目的**：通过共识机制严格筛选候选经验。
        -   **方法**：候选经验被提交给由原始执行智能体构成的“验证组”进行评判。验证采用**严格的默认拒绝策略**：
            -   **一致性通过**（全体同意）：写入共享记忆库。
            -   **部分通过**（部分同意）：写入生成该经验的蒸馏智能体对应的**私有记忆库**。
            -   **未通过**：直接丢弃。

-   **算法流程的文字说明**：
    -   **经验构建**：`任务 -> [Execute: 多异构智能体并行执行 -> 多条候选轨迹] -> [Distill: 第三方智能体对比分析 -> 候选经验] -> [Verify: 执行组共识验证 -> 通过/部分通过/丢弃] -> [写入共享/私有记忆库，更新能力矩阵]`。
    -   **推理时使用**：`新任务 -> [能力矩阵匹配 -> 选择最佳解算器] -> [层次化检索：先搜索共享库，不足则搜索选定解算器的私有库] -> [将检索到的记忆拼接到任务上下文中 -> 指导推理]`。
