## 摘要
提升视觉语言模型（VLM）在视觉推理上的能力通常需要重新训练或手工设计的提示和工具。我们提出Dynamo，一种无需训练的框架，能够在不更新任何权重的情况下适应冻结的VLM。在一个小的标注训练子集上，代理检查自身正确和错误的尝试，并演化出两种互补能力：用于认知瓶颈的可复用推理技能，以及用于感知瓶颈的可执行视觉工具。每个生成的工具都配有一个指定何时调用它的技能，两种能力类型都积累在一个持久库中。在四个视觉推理基准和五个VLM骨干网络上，Dynamo在所有20个模型-基准设置上提升了直接推理性能（平均+5.6准确率）。当工具集预先给出时，该框架学习何时调用每个工具，并且每一步的工具选择在所有测试的骨干网络上都有改进。针对特定任务强化学习（VTool-R1, DeepEyes），Dynamo以极少的计算量填补了65-99%的RL差距，并且在可用时可以与RL相加组合。

## Abstract
Improving vision-language models (VLMs) on visual reasoning typically requires retraining or hand-designed prompts and tools. We present Dynamo, a training-free framework that adapts a frozen VLM without any weight updates. On a small labeled training subset, the agent inspects its own correct and incorrect attempts and evolves two complementary capabilities: reusable reasoning skills for cognitive bottlenecks, and executable visual tools for perceptual ones. Each generated tool is paired with a skill that specifies when to invoke it, and both capability types accumulate in a persistent library. Across four visual reasoning benchmarks and five VLM backbones, Dynamo improves direct inference on all 20 model--benchmark settings (avg. +5.6 acc). When the tool set is given in advance, the framework learns when to call each tool, and per-step tool choice improves on every tested backbone. Against task-specific RL (VTool-R1, DeepEyes), Dynamo closes 65--99% of the RL gap at a fraction of the compute, and combines additively with RL when available.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：让冻结的VLM在一个小标注训练子集上，通过自我检查正确与错误尝试，自动生成两种互补能力：**技能**（结构化Markdown SOP，用于认知瓶颈）和**工具**（Python程序，用于感知瓶颈）。每个工具配有一个配对技能（mastery SOP）来指定何时调用，两者积累在持久库中。推理时根据问题检索相关技能和工具。
- **关键技术细节**：
  - **演化循环**（每轮三个步骤）：
    1. **诊断**：当前代理尝试训练子集（采样包含正确和错误案例），分析器判断瓶颈类型（认知还是感知）并决定生成技能、工具或两者。
    2. **探索**：生成器提出M个候选技能+工具组合。
    3. **验证与提升**：评估每个候选在训练集上的准确率，只提升那些严格改进当前能力的组合，并确保不回归。

  - **在线适应**：通过滚动窗口检测查询分布漂移，当主导任务族变化时自动触发新的演化迭代，更新库。

## 7. 优点

- **训练免费**：无需梯度更新，大幅降低计算和数据需求，适用于大模型适配场景。
- **动态自动化**：代理自主诊断错误原因，生成并验证技能和工具，无需人工干预。
- **可解释性**：生成的技能和工具是显式的（Markdown和Python），便于理解和调试。
- **模块化与可组合**：技能和工具积累在库中，可与RL等现有方法组合，产生增益。
- **鲁棒性**：验证和提升机制确保不会因新能力而退化，支持在线适应分布漂移。
- **广泛的实验验证**：覆盖多骨干、多基准、多种对比，消融充分，结论可靠。

## 8. 不足与局限

- **依赖正确性信号**：诊断和演化需要每个案例的正确性信号（实验中用标注真实标签），在完全无监督的生产环境中不可用。论文承认需要人工审核或LLM验证器作为替代，但未完全解决。
- **诊断受骨干自我反思能力限制**：分析器和生成器使用同一个冻结VLM，如果骨干自身不强，诊断可能不准确，导致演化缓慢或库质量低。
- **基准范围有限**：仅评估了图像型VLM的视觉推理，未涵盖3D空间推理、医学影像等其他模态。认知/感知分类和工具接口的迁移性未知。
- **多候选探索的规模**：默认M为个位数（单数字预算），论文提到更大范围的探索留待未来。
- **计算成本虽低但仍有API调用**：对每个基准需要几千次API调用，仍有一定成本，但远低于RL。
- **消融中“无持久化”机制探针不完整**：由于候选被拒绝，未能在30案例子集上有效测试持久化的影响，跨案例重用证据较弱。
