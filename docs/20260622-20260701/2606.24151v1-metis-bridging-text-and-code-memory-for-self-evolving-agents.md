## 摘要
自进化智能体通过从过去执行中蒸馏经验并在未来任务中重用这些经验来随时间改进。现有系统将此类经验表示为注入到智能体上下文中的自然语言文本，或表示为可调用工具暴露的代码。然而，这两种表示之间的选择通常在设计时做出，而非从经验本身的特征推导而来，导致对二者之间权衡的理解不足。我们提出了首个对照研究，在相同的经验集上隔离文本记忆和代码记忆。我们的结果表明，两种形式在构建成本、执行效率和可迁移性方面表现出互补的权衡，以至于单独任何一种表示都不足够。基于这些发现，我们提出了Metis，一个基于分层双表示记忆的自进化智能体系统。Metis将文本经验组织为执行计划、环境事实和常见陷阱，并选择性地将重复出现的计划固化为经过验证的可调用工具。这种设计结合了文本记忆的广泛适用性和代码记忆的执行效率，同时仅在重复使用合理时才产生工具生成成本。我们在AppWorld（一个具有挑战性的交互式智能体基准）上评估了Metis。结果表明，与ReAct相比，Metis将任务准确率提高了高达20.6%，同时将执行成本降低了高达22.8%。与代表性的自进化智能体系统相比，Metis在准确率、执行效率和记忆构建成本之间持续实现了更好的平衡。

## Abstract
Self-evolving agents improve over time by distilling experience from past executions and reusing it in future tasks. Existing systems represent such experience either as natural-language text injected into the agent context or as code exposed as callable tools. However, the choice between these representations is typically made at design time rather than derived from the characteristics of the experience itself, leaving the trade-offs between them poorly understood. We present the first controlled study that isolates text memory and code memory over an identical set of experiences. Our results show that the two forms exhibit complementary trade-offs in construction cost, execution efficiency, and transferability, such that neither representation alone is sufficient. Guided by these findings, we propose Metis, a self-evolving agent system built on a hierarchical dual-representation memory. Metis organizes textual experience into execution plans, environment facts, and common pitfalls, and selectively crystallizes recurring plans into validated callable tools. This design combines the broad applicability of text memory with the execution efficiency of code memory while incurring tool-generation cost only when justified by repeated reuse. We evaluate Metis on AppWorld, a challenging benchmark for interactive agents. The results show that Metis improves task accuracy by up to 20.6% over ReAct while reducing execution cost by up to 22.8%. Compared with representative self-evolving agent systems, Metis consistently achieves a better balance between accuracy, execution efficiency, and memory-construction cost.

## 2. 论文提出的方法论

### 核心思想
- **层次化双表示记忆**：基础是结构化文本记忆（低承诺、低成本、易迁移），上层是代码记忆（高执行效率但构建成本高、可迁移性差）。代码记忆仅从文本记忆中的执行计划提升，且必须满足重复使用阈值。
- **三大关键机制**：
  1. **差异化文本反射**：将文本经验分为三类——执行计划（可提升为代码）、环境事实（任务无关约束）、常见陷阱（失败模式）。反射器是智能体自身，可调用环境工具进行验证，并进行合并/更新/失效维护。
  2. **模式感知代码生成**：仅当执行计划的候选缓冲区大小达到阈值θ时触发代码生成。代码生成器只使用执行计划体和相关的任务查询（不包括原始轨迹），生成小且可组合的辅助工具，并通过依赖闭合、编译检查和沙箱验证后才加入代码库。
  3. **记忆管理器**：联合选择文本记忆和代码工具，使用嵌入检索和LLM过滤，并自动进行依赖闭合，确保工具注入的完整性。

### 关键技术细节
- **反射路径与执行路径分离**：反射在完成轨迹后异步进行，不增加任务服务延迟。
- **渐进式轨迹披露**：反射器先接收压缩视图，可按需扩展特定步骤，提高反射效率。
- **失效机制**：过时条目通过失效位标记，不原地编辑，保持一致性。
- **代码生成条件**：仅当计划被选中足够多次（|B(p)| ≥ θ），且使用两条以上结构不同的查询，任务变化元素作为参数。

### 算法流程（文字描述）
1. 对每个任务i，Memory Manager检索并联合选择文本记忆和代码工具。
2. 执行器在所选记忆条件下执行任务，产出轨迹τi。
3. 双重反射：
   - 首先，如果选中的执行计划累计重用证据达到阈值，触发模式感知代码生成（Codify pipeline），仅使用查询（不包含轨迹），生成工具并验证后加入代码库。
   - 然后，差异化文本反射处理轨迹，更新文本记忆（新建、合并、失效）。
4. 更新后的记忆用于后续任务。

## 6. 主要结论与发现

1. **文本与代码记忆互补**：文本构建成本低（约2.5倍更少的LLM轮次）、可迁移性好（流式设置下仅下降5.6个点，而代码下降22.3点）；代码执行效率高（相同任务节省约54% token和46%步骤）。
2. **Metis整体优越**：在Official split上准确率达60.1%（相比No Memory 51.8%提升8.3点），执行token减少13.5%，反射成本低于SkillX；在Resampled split上准确率66.1%（提升11.3点），执行token减少22.8%。
3. **重复性门控至关重要**：Eager变体（无门控）导致构建成本增加47%，准确率下降3.5点，且工具调用率低（41% vs 56%），工具过拟合单个任务。
4. **双重表示优于单一表示**：Text-only和Code-only均不如完整Metis，混合增益来自两者交互（计划驱动代码生成，代码改进计划）。
5. **代码工具实际被使用**：Metis的94%代码工具在测试中被至少调用一次（而SkillX的84.1%仅被注入上下文）。

## 7. 优点

1. **首次系统研究记忆表示**：通过受控对照实验揭示了文本与代码的互补权衡，为后续设计提供理论指导。
2. **层次化设计与选择性提升**：避免将所有经验代码化，只在重复性验证后才进行高成本代码生成，兼顾效率和泛化性。
3. **反射与执行解耦**：反射在异步路径进行，不增加在线服务延迟，实际部署友好。
4. **多种安全保障**：轨迹不被代码生成使用、沙箱验证、依赖闭合检查、失效机制，降低错误传播风险。
5. **经验组织精细**：文本记忆细分为三类（计划、事实、陷阱），各自有不同的检索和使用模式，避免混杂。
6. **在挑战性基准上表现突出**：同时提升准确率、降低执行成本，且记忆构建成本可控。
