## 摘要
对于在多步交互环境中的大语言模型智能体，一个关键挑战是如何有效利用累积的交互经验。现有工作通常将这种经验的双重用途分开：将其作为自然语言规则保留在模型外部以供后续提示使用，或者使用轨迹和反馈来更新模型参数。前者易于解释，但可能随着策略的演化而不同步；后者可以更广泛地改进策略，但在稀疏奖励设置中对局部错误的纠正有限。我们提出了面向大语言模型智能体的经验规则与策略联合学习方法（JERP），该方法从相同的交互轨迹中更新长期经验规则池和策略。在决策时，JERP检索任务相关规则，并将这些规则与交互历史一起作为智能体的条件。每回合结束后，它使用收集的轨迹来优化策略，并通过将当前展开与参考成功轨迹进行比较来修订规则池。这种耦合使规则池与不断演化的策略保持一致，同时允许稳定有效的行为逐渐被吸收到模型本身中。在AlfWorld和WebShop上的实验表明，JERP在复杂交互任务的决策性能上取得了持续改进。

## Abstract
For LLM agents in multi-step interactive environments, a key challenge is to make effective use of accumulated interaction experience. Existing work has typically separated two uses of such experience: keeping it outside the model as natural-language rules for later prompting, or using trajectories and feedback to update the model parameters. The former is easy to interpret but can fall out of sync with the evolving policy; the latter improves the policy more broadly but provides only limited correction for local mistakes in sparse-reward settings. We present Joint Learning of Experiential Rules and Policies for LLM Agents (JERP), which updates a long-term experiential-rule pool and the policy from the same interaction trajectories. At decision time, JERP retrieves task-relevant rules and conditions the agent on them together with the interaction history. After each episode, it uses the collected trajectories both to optimize the policy and to revise the rule pool by comparing current rollouts with reference successful trajectories. This coupling keeps the rule pool aligned with the evolving policy while allowing stable and effective behaviors to be gradually absorbed into the model itself. Experiments on AlfWorld and WebShop show that JERP yields consistent gains in decision performance for complex interactive tasks.

- **研究动机**：在多步交互环境中（如网页导航、具身交互），LLM智能体需要有效利用累积的交互经验来改进未来决策。现有方法通常将经验用于两种分离的途径：  
  - **外置规则**：将经验转化为自然语言规则，存储并检索到提示中（如Reflexion、ExpeL、AutoGuide）。优点是可解释、可编辑；缺点是规则可能随着策略参数的更新而变得过时，与演化中的策略不匹配。  
  - **策略参数更新**：使用交互轨迹和反馈通过强化学习（如GRPO、RLOO）直接更新模型参数。优点是全局行为改进；缺点是稀疏奖励环境下局部错误可能得不到及时纠正，且缺乏可检查的显式知识。  
- **核心问题**：如何让同一批交互数据同时支持显式规则维护和策略参数优化，使两者相互增强，而非割裂。  
- **整体含义**：JERP提出规则池与策略的**联合学习**，决策时检索规则作为上下文，每轮后用轨迹同时更新策略和规则池，实现规则与策略的动态对齐，提升在复杂交互任务上的决策效果。

## 2. 论文提出的方法论

### 2.1 核心思想
- 维护一个**长期经验规则池**（per-task），每个规则包含文本内容和效用分数。  
- 每轮互动开始时，从池中按效用分数降序选取**top-k规则**作为工作规则集，与任务描述和交互历史一起构成prompt。  
- 采样一个**轨迹组**（N条轨迹）：旧策略在给定工作规则集下与环境交互。  
- 轨迹组被用于**两个更新阶段**：  
  1. **策略参数更新**：基于GRPO目标，使用组内相对优势（标准化后的轨迹奖励）进行策略梯度优化。  
  2. **规则池更新**：通过对比当前轨迹与参考成功轨迹，生成结构化规则编辑操作（ADD, EDIT, UPVOTE, DOWNVOTE, MERGE），更新规则文本和效用分数，并移除低于阈值的规则。

### 2.2 关键技术细节
- **规则存储与检索**：每个任务d对应规则池K(d) = {(z_i, s_i)}，s_i为效用分数。决策时取top-k规则（k为超参数）。工作规则集在单回合内固定，避免上下文波动。  
- **策略优化**：采用GRPO（Group Relative Policy Optimization）。对轨迹组{τ_i}计算标准化优势A_i = (R(τ_i)-μ_d)/(σ_d+δ)，所有时间步共享同一优势。更新目标为裁剪的PPO目标加KL散度正则化。  
- **规则更新（对比式反思）**：  
  - 输入：当前规则池K(d)、当前轨迹组T_d、参考成功轨迹集T^+(d)（从之前训练中累积的奖励>0的轨迹）。  
  - 使用一个固定参数的LLM（冻结的反思模型）生成操作列表O(d)，包含五种类型：  
    - **ADD(z)**：添加新规则（初始分数s0）  
    - **EDIT(q, z)**：修改规则q的文本（保留原分数）  
    - **UPVOTE(q)** / **DOWNVOTE(q)**：增加/减少规则q的效用分数（固定增量δ+, δ-）  
    - **MERGE(Q, z)**：合并一组功能重叠的规则为一条（新分数取原最大分数）  
  - 应用操作后，若规则分数低于阈值则移除。规则池设有容量上限，超出时移除低分规则。  
- **参考成功轨迹集**：从之前训练中累计的奖励>0的轨迹，无人类示范或测试数据。若无成功轨迹则仅依赖当前轨迹组更新。

## 6. 主要结论与发现

- **JERP在整体成功率上优于所有基线**：AlfWorld总体成功率61.5% vs GRPO 57.8% vs RLOO 48.7% vs Reflexion 21.8% vs ReAct 12.8% vs Vanilla 4.1%；WebShop成功率64.1% vs GRPO 56.2% vs RLOO 57.8% vs Reflexion 21.9% vs ReAct 11.3% vs Vanilla 5.2%。  
- **增益集中于长序列、多约束的任务**（如Clean, Heat, Cool, Pick2），因为这些任务反复出现的中期错误可通过规则池中的局部修正得到缓解；而简单任务（Pick, Look）上GRPO略优，说明规则池在简单场景下可能增加冗余。  
- **动态规则池更新至关重要**：固定初始规则池的消融变体后期性能增长停滞，验证了规则池与策略演化的同步必要性。  
- **联合学习模式有效**：规则池保留了可解释的局部知识，同时策略参数实现全局改进，两者协同提升。

## 7. 优点

1. **方法创新性**：首次将外置规则学习与策略参数优化纳入同一训练循环，解决了规则与策略失配的问题。  
2. **规则池的可维护性**：设计了完整的规则生命周期管理（添加、修改、投票、合并、剪枝），使用LLM自动生成结构化生成，实现规则池的动态精简和适应。  
3. **实验设计严谨**：在两个困难基准上对比多种代表性基线，消融实验直接验证核心机制，训练曲线展示动态过程。  
4. **可解释性**：规则以自然语言存储，能理解智能体为何采取某些行为，便于错误分析和人工干预。  
5. **效率**：使用LoRA进行参数高效微调，避免昂贵全量微调；规则更新使用冻结的LLM，额外开销可控。

## 8. 不足与局限

1. **规则检索的粗粒度**：当前只按效用分数取全局top-k，缺乏对当前具体任务实例或状态的细粒度相关性检索（论文自己也指出这一点属于实现选择，可进一步改进）。  
2. **规则更新依赖固定LLM**：反思模型的生成质量直接影响规则有效性；若基座模型变化，反思模型可能需要同时更新。  
3. **仅在一个基座模型上实验**：未说明具体基座模型（可能是Llama-2或类似），也未测试不同大小或家族的模型，结论的泛化性有待验证。  
4. **环境局限性**：仅在文本交互环境（AlfWorld, WebShop）上测试，未在视觉-语言或多模态环境、机器人控制等更复杂场景下验证。  
5. **规则校准与副作用风险**：规则池维护可能引入冗余或错误规则，导致智能体过度依赖规则而忽略新情况；MERGE操作需谨慎，可能丢失有用差异。  
6. **计算与内存开销**：规则池需存储每条完整文本，且在prompt中加入多条规则增加tokens消耗；规则更新每步需调用一次外部LLM（反思模型），可能拖慢训练速度。  
7. **缺乏与多种强化学习方法的更系统对比**：仅对比了RLOO和GRPO，未对比PPO_full, DPO多轮版本, GiGPO（论文相关工作中提到的）等。
