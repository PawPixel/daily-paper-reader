## 摘要
自动化Alpha挖掘保持评分函数固定，并在此之上改变搜索算法。针对固定评分器的收敛搜索会过度拟合评分器无法惩罚的任何因素，这是样本外泛化差距的主要原因。我们将评分函数视为与Alpha因子一同的搜索产物，并研究何种条件能使这种联合搜索成立。密封联合搜索（SJS）是一个框架：它包含一组关于自主发现系统中信息流的结构条件，防止联合搜索陷入自我确认，同时保持评估器密封。这些条件涵盖角色分解、类型化角色间通信、来源密封读取、版本化存储以及基板局部提升。Agora对SJS进行了实证测试：五类LLM智能体通过三个通道通信，演化出八个技能库，其中Alpha库基于AlphaGen算子构建。三个评估器撰写报告并汇总为一份简报，保留分歧而非投票。我们在CSI 1000上运行Agora 100轮，并在一个对所有LLM输入密封的91天2026年持有期上进行评估。Agora的持有期夏普比率为+1.87；最佳基线在有利种子下为+1.334，跨种子均值为-0.755。将Agora的两个指标预加载到冻结库消融中仅恢复+2.25夏普差距中的+0.40，而添加无库演化的PPO则使差距恶化。这两个指标是涌现出来的而非设计的。注意事项：单种子运行，信号集中于空头，旨在多空操作。

## Abstract
Automated alpha mining holds the scoring function fixed and varies the search algorithm over it. A search that converges against a fixed scorer overfits whatever the scorer cannot penalize, a primary cause of the out-of-sample generalization gap. We treat the scoring function as a search artifact alongside the alpha factors and study what conditions make this joint search admissible. Sealed Joint Search (SJS) is a framework: a set of structural conditions on information flow in an autonomous-discovery system that prevent joint search from collapsing into self-confirmation while keeping the evaluator sealed. Conditions cover role decomposition, typed inter-role communication, provenance-sealed reads, versioned stores, and substrate-local promotion. Agora tests SJS empirically: five LLM agent classes communicate via three channels, evolving eight skill libraries, with alpha libraries built on AlphaGen operators. Three evaluators write reports aggregated into one brief, carrying forward disagreement instead of voting. We run Agora for 100 rounds on CSI 1000 and evaluate on a 91-day 2026 holdout sealed from all LLM inputs. Agora achieves holdout Sharpe +1.87; best baseline +1.334 at favorable seed and -0.755 cross-seed mean. Pre-loading Agora's two metrics into a frozen-library ablation recovers only +0.40 of the +2.25 Sharpe gap, and adding PPO without library evolution worsens the gap. The two metrics emerge rather than being designed. Caveats: single-seed run, short-side concentrated signal, intended for long-short.

## 论文结构化总结

### 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：自动化Alpha挖掘（以及更广泛的自主发现）中，传统方法将评分函数固定不变，仅变化搜索算法。这种范式导致搜索算法会过度拟合评分函数无法惩罚的盲点，从而产生严重的样本外泛化差距。即使改进搜索算法，也无法解决评分函数本身被过拟合的问题。
- **整体含义**：作者提出应从“搜索算法优化”转向“信息拓扑结构设计”。将评分函数也视为可搜索的工件，使信号（f）和评分函数（g）能够联合进化，同时保持外部评估器密封，避免系统陷入自我确认。这一框架旨在提升泛化能力，并可能迁移到定理证明、程序合成等其他自主发现领域。

### 2. 方法论：密封联合搜索（SJS）框架与Agora系统

- **核心思想**：将搜索过程视为一个有向图，节点包括生产者、评估者、存储，边是带类型的信息通道。泛化性能取决于全局图拓扑，而非单个节点的更新规则。SJS定义了五个信息流条件：
  - **P1 - 带类型非对称传递**：每条边携带内容类型和接收角色；发送方不能控制接收方看到什么，由基板在边缘计算不同载荷。
  - **P2 - 有界容量跨轮边**：跨轮边容量由基板限定，防止智能体间无限信息积累。
  - **P3 - 来源决定读取范围**：每条记录有数据段来源标签；角色只能读取与其授权内容类型和来源匹配的记录。持有段对提案角色不可见。
  - **P4 - 技能存储作为进化场所**：持久化版本化工件存储，每个存储由唯一角色拥有，其他角色不能写入。存储中的工件有 builtin / trial / accepted / rejected 四种状态。
  - **P5 - 闭包诱导状态转换**：存储状态的转换由基板根据自身产生的记录计算，不依赖外部评估器，转换规则基于内部结果变量（如训练段夏普比率）。
- **边界条件**：外部评估器在运行前固定，对所有角色不可访问。
- **Agora系统**：SJS的A2A（智能体对智能体）实例化：
  - 5个角色（研究报告、Alpha挖掘者、评估挖掘者、两个评估面板各3个实例），共9个独立LLM客户端（claude-sonnet-4-6）。
  - 3个通信通道：通道A（轮内定向建议）、通道B（跨轮有界简报）、通道C（LLM Wiki，按角色过滤）。
  - 8个技能库：算子、RL网络、RL算法、奖励由Alpha挖掘者拥有（基于AlphaGen）；度量库、主题库、评估准则库等分布给其他角色。度量库初始仅4个内置指标，是唯一观察到显著进化的库。
  - 提升规则：当度量分数与训练段夏普的|ρ| > 0.3时自动提升为 accepted；|ρ| < 0.05时降级；也可通过智能体发起的评估面板批准。

### 3. 实验设计

- **数据集**：CSI 1000（中证1000成分股）动态面板，2014-10-17至2026-05-27。数据来源：RiceQuant，包含调整后OHLCV、可交易掩码、涨跌停掩码、次日开盘价。
- **时间划分**：训练段2014-10至2019-12（1277交易日），测试段2020-01至2025-12（1461交易日），持有段2026-01至2026-05（91交易日）。所有方法使用相同划分。
- **基线方法**（7种）：
  - B1: 遗传规划（gplearn，基于AlphaGen参考实现）
  - B2: AlphaGen纯PPO（种子42，另加种子0,1用于跨种子分析）
  - B3: 单LLM一次性提示
  - B4: 单LLM迭代（10轮，每轮反馈Top3/Bottom3）
  - B5: Alpha101（WorldQuant Aplha 1-30子集）
  - B6: 冻结库消融（禁用所有技能库进化，保留Agora架构其余部分）
  - B7: 随机搜索（3000随机表达式树）
- **评估**：外部回测（10分位排序，5天再平衡，9bps单边成本）。主要指标：持有期组合夏普比率、组合IC、分位单调性、年化收益率。统计检验：Newey-West HAC t检验（滞后5）、两样本bootstrap、Mann-Whitney U检验。

### 4. 资源与算力

- 文章中明确说明：Agora完整运行（100外轮）使用了约60 GPU-小时（RTX 5090）加上4500次LLM调用（claude-sonnet-4-6）。每轮约45次LLM调用，100次PPO中继（每次15000时间步）。
- 硬件：AMD Ryzen 9 CPU + NVIDIA RTX 5090 GPU，单机环境。
- 各基线运行时间：B2纯PPO约5小时（RTX 5090）；B6冻结库5轮约1.5小时/轮；其他基线未提供精确时间，但总体算力开销较大（尤其是多智能体系统）。
- 跨种子复制：B2额外运行2个种子（0,1），B6额外运行1个种子（0）；Agora仅单种子（受算力限制，作者说明预算不足以运行第二种子）。

### 5. 实验数量与充分性

- **实验数量**：主要对比7种基线，加上3种Agora内部消融（B6+增强内置指标、B6+PPO中继、B6+增强+更多轮），以及滚动窗口分析（6个历史6月窗口）。总计约12+种配置。
- **充分性**：
  - 较充分：覆盖了传统符号搜索、深度RL、纯LLM、随机搜索、消融实验；消融逐步分解了SJS各组件贡献；统计检验（NW HAC）对主要假设进行了推断。
  - 不充分之处：

### 6. 主要结论与发现

- **性能领先**：Agora持有期组合夏普+1.87，最佳基线B2（单种子+1.334，跨种子均值-0.755）。Agora在组合IC（+0.089）、分位单调性（+0.285）和年化多空收益（+48.4%）上也领先。
- **涌现而非设计**：度量库中仅两个度量被提升：单调性分数v1（21轮，智能体发起）和过度回撤惩罚v1（50轮，自动提升）。它们都是标准量化实践，但由系统从稀疏内置集通过SJS涌现产生，无人工指定目标。
- **消融分解**：
  - 冻结库消融（B6）夏普-0.379，与Agora差距+2.25，为联合贡献上界。
  - 预加载两个度量至冻结库仅恢复+0.40夏普（<20%），说明静态知识不替代持续进化。
  - 仅添加PPO中继（无库进化）导致夏普-1.18，比B6更差：不对称升级搜索程序而无协同进化评分函数有害。
  - 更多轮次下冻结库表现更差（B6+增强20轮夏普-1.96），排除轮数解释。
- **信号特征**：信号集中于空头（底层分位年化-27.39%，顶层+6.46%），适合多空操作；长持夏普在多成本下保持正，长端在5倍成本下转负。
- **滚动窗口**：同一30个Alpha在7个历史6月窗口中7/7为正多空夏普，2026持有段位于中位数，表明一定稳健性但测试段有软监督污染。

### 7. 优点

- **方法论创新**：提出SJS框架，将设计焦点从搜索算法转移到信息拓扑结构，概念清晰，可迁移。首次将评分函数作为可搜索工件并给出形式化条件。
- **实验细粒度**：消融实验系统分解了不同组件贡献，特别是揭示“不对称升级有害”的发现具有重要启示。
- **泄漏审计**：详细说明了五条潜在泄漏路径及其密封措施（训练/测试/持有段分离、分类桶反馈、禁止数值传递等），透明度高。
- **统计严谨性**：使用Newey-West HAC处理序列相关，报告多种子结果，不隐藏负面发现（如PPO中继单独的负效果）。
- **开源承诺**：计划发布完整代码、基线、日志，可复现性强。

### 8. 不足与局限

- **单种子运行**：Agora仅一个种子，种子方差未知。即使Agora点估计强于B2跨种子均值，但跨种子比较是间接的，需更多种子验证。
- **统计显著性有限**：在91天持有期上，NW HAC仅对B1和B7显著；与最强基线B2的对比在统计上不显著，置信区间极宽。
- **消融不完整**：
  - 缺少“单智能体+进化”基线（直接检验F1角色分解的必要性）。
  - B6+PPO中继和B6+增强20轮均为单种子，结论初步。
  - 8个库中仅度量库进化，其他7个库未动，限制了“自进化”的普遍性。
- **样本外污染**：测试段通过分类桶反馈对度量挖掘者产生软信号，2026持有段虽完全干净但仅5个月，时间跨度短。
- **信号集中空头**：长持部分表现一般（跑输基准），策略仅适合多空，限制了应用场景。
- **领域局限**：仅在CSI 1000中国A股上测试，未验证迁移到其他市场或自主发现领域。
- **外部评估器依赖**：P5使用的内部结果变量（训练段夏普）与最终目标（持有段夏普）之间存在相关性假设，该假设的强度影响进化信号质量，但未充分讨论。
- **计算成本高**：Agora需要60 GPU小时+4500 LLM调用，对于学术实验室可能过高，限制了复制和扩展。
