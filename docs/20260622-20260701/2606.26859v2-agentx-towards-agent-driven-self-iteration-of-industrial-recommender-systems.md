## 摘要
推荐算法迭代正从一种工匠式的、依赖工程师的过程转向工业化的研究循环，但这一转变仍受到结构性执行瓶颈的阻碍：从构思到上线的周期仍然依赖人类工程师来生成假设、修改生产代码、启动A/B实验并归因线上结果。因此，创新与人力成正比增长，而不是随着证据、计算和积累的实验知识而复合增长。我们提出AgentX，一个生产部署的多智能体系统，从根本上重构了这一生产函数。AgentX作为一个自我进化的开发引擎运行：它以任何手动工作流程都无法维持的规模和速度自主生成、实现、评估并从推荐实验中学习。该系统在闭环中编排四个紧密耦合的阶段。头脑风暴智能体将来自历史实验、系统架构、数据分析和外部研究的证据综合成按优先级排序的可执行提案。开发智能体通过基于仓库的生成和多维可靠性验证，将每个提案转化为生产就绪的代码。评估智能体通过带有护栏否决的A/B判断进行安全在线部署，将成功和失败都转化为结构化的知识资产。然后，一个驾驭进化层（SGPO）将执行轨迹提炼为语义梯度更新，不断锐化智能体本身——使系统不仅自动化，而且自我改进。

## Abstract
Recommendation algorithm iteration is moving from an artisanal, engineer-bound process toward an industrialized research loop, but this transition remains blocked by a structural execution bottleneck: the idea-to-launch cycle still depends on human engineers to generate hypotheses, modify production code, launch A/B experiments, and attribute online results. Innovation therefore scales linearly with headcount rather than compounding with evidence, compute, and accumulated experimental knowledge. We present AgentX, a production-deployed multi-agent system that fundamentally restructures this production function. AgentX operates as a self-evolving development engine: it autonomously generates, implements, evaluates, and learns from recommendation experiments at a scale and pace that no manual workflow can sustain.   The system orchestrates four tightly coupled stages in a closed loop. A Brainstorm Agent synthesizes evidence from historical experiments, system architecture, data analysis, and external research into ranked, executable proposals. A Developing Agent translates each proposal into production-ready code through repository-grounded generation and multi-dimensional reliability verification. An Evaluation Agent conducts safe online rollout with guardrail-vetoed A/B judgment, converting both successes and failures into structured knowledge assets. A Harness Evolution layer (SGPO) then distills execution trajectories into semantic-gradient updates that continuously sharpen the agents themselves -- making the system not merely automated, but self-improving.

**核心思想**：通过解耦推荐系统研发的生命周期，构建一个由四个紧密耦合阶段组成的**闭环循环**（Closed-loop）。这个循环将人类的角色从“执行者”提升为“决策者和系统设计者”。

**四个核心阶段及关键技术细节**：

1.  **头脑风暴智能体 (Brainstorm Agent)**
    - **功能**：将模糊的优化意图转化为少量、可执行、有实证支持的实验提案。
    - **关键技术**：
        - **歧义消除 (Ambiguity Resolution)**：将不确定的用户请求规范化，形成明确的任务边界。
        - **有界提案探索 (Bounded Proposal Exploration)**：批量化生成提案，并根据成熟度（可实施/需验证/远期目标）进行分层，避免早期过早承诺。
        - **证据加权提案生成 (Evidence-Weighted Proposal Generation)**：从四个知识库（实验知识库、系统知识库、数据分析、模型研究）中检索证据，并根据提案类型动态分配权重，进行打分排序。评分公式为：
            \[
            S(c|q) = \lambda_o O(c, q) + \lambda_b B(c) + \lambda_f F(c) + \lambda_h H(c) + \lambda_e E(c|q) - \lambda_r R(c)
            \]
            其中 `O` 为目标对齐度，`B` 为业务有效性，`F` 为可行性，`H` 为交接完整性，`E` 为综合证据得分，`R` 为风险惩罚。

2.  **开发智能体 (Developing Agent)**
    - **功能**：将经过审批的实验提案转化为经过验证、安全的生产代码。
    - **关键技术**：
        - **分支处理**：并行处理在线策略（代码修改）和离线模型（训练实验）两大任务。
        - **基于仓库的代码生成 (Repository-Grounded Code Generation)**：通过查询项目特定的知识库和确定性工具（如模式查询工具、DSL检查器），防止属性幻觉（inventing fields）和DSL滥用。
        - **面向验证的实现循环 (Verification-Oriented Implementation Loop)**：代码生成后，通过准确性循环（与计划对比）和Dryrun管线（编译和集成检查）进行多轮验证。
        - **质量评分 (Quality Scoring)**：使用8维度的加权得分（`Q_code`），重点惩罚属性幻觉、人类干预和正确性循环开销等破坏自动化可靠性的问题。

3.  **评估智能体 (Evaluation Agent)**
    - **功能**：通过安全的在线部署和A/B实验，将用户真实反馈转化为可靠的、可复用的奖励信号，并管理实验结果。
    - **关键技术**：
        - **安全部署与流量分配**：确保实验在正确的业务域、流量桶中运行，遵循灰度发布等安全规则。
        - **带护栏否决的A/B判断**：使用CUPED等方差缩减方法进行指标分析。护栏设计遵循三个原则（业务范围化、复合经济交换指标、作为关注信号而非绝对阻断），防止误判。
        - **负结果资产化 (Negative-Result Assetization)**：将失败的实验（DISCARD/Inconclusive）进行归因分析并结构化存储，成为未来迭代的“避坑指南”，防止重复犯错。

4.  **驾驭进化层 (Harness Evolution)**
    - **功能**：通过分析累积的执行轨迹，逆向优化前三个智能体本身的“驾驭”（指智能体的指令、验证规则、输出契约等），使整个系统不断完善。
    - **核心技术：语义梯度提示优化 (Semantic-Gradient-based Prompt Optimization, SGPO)**
        - **SGPO-I (基于对话轨迹)**：从用户会话的失败案例中提取“语义梯度”（结构化诊断报告），然后使用一个优化智能体据此更新目标智能体的“驾驭”规范。新规范需通过“配对回放”（Paired Replay）的严格验证（新旧版本在同一任务上对比）才能正式生效。
        - **SGPO-II (基于代码合并请求)**：从历史成功的代码提交中提取案例，通过回放，让开发智能体从失败中学习，优化其代码生成规则、模式记忆和确定性预检查，从而提升代码质量和可维护性。
        - **模型研究探索**：系统通过三个阶段的循环（文献复现→模块消融→跨论文组合）来自主探索和发现新的模型架构，并通过内存机制（基于实验结论的论文级/模块级剪枝）和饱和检测来保证搜索的效率和有效性。

#### 7. 优点

- **完整性**：构建了从想法到上线再到自我进化的最完整闭环，是理论到实践的典范。
- **真实回报**：使用线上A/B测试作为最终奖励信号，将系统优化直接与真实的商业价值对齐，这是最核心的优势。
- **自进化能力**：通过SGPO让系统自身不断改进，是迈向真正通用人工智能研发助手的标志性特征，解决了现有工作缺乏长期积累的问题。
- **负结果资产化**：将实验失败也视为宝贵的知识资产，避免了重复犯错，体现了系统长期的“记忆力”和“学习能力”，是工程智慧的重要体现。
- **安全与稳健的设计**：采用灰度发布、护栏机制、配对回放验证等方法，确保了自动化系统在工业级复杂环境中的安全性和可靠性。

#### 8. 不足与局限

- **应用场景与泛化性**：所有实验都基于快手平台。系统对应用户、架构、DSL和商业规则有很强的耦合性，迁移到其他公司的不同推荐系统时，需要大量的适配工作。系统的知识库（System KB, Experiment KB）需要为新系统从头构建。
- **实验成本与资源透明度**：虽然论文展示了效率的大幅提升，但并未披露AgentX系统本身的运行和维护成本（如LLM API调用费用、GPU训练Agent自身模型的开销）。这种“自动化”对计算资源的消耗可能非常巨大。
- **系统复杂性**：一个包含多个专业Agent、知识库、进化层的系统，其自身的稳定性和可调试性是一个巨大的挑战。出现bug时，排查是智能体逻辑错误、知识库错误还是进化层问题可能非常困难。
- **“护栏”可能过于保守，导致假阴性**：论文指出，其保守的决策策略（Guardrail Veto）是为了安全。但这也可能错误地扼杀掉一些虽然有风险但具有巨大长期价值的创新性实验。虽然论文声称通过设计（如使用复合指标）来缓解，但“完美”的护栏在理论上是做不到的。
- **LLM的幻觉与不确定性**：尽管系统通过多种机制（如代码生成时的确定性检查）来规避LLM的缺陷，但LLM作为核心推理引擎，其固有的不确定性和潜在的幻觉仍然是系统的根本风险点。SGPO的进化过程也可能引入新的偏差。
