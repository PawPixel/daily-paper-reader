## 摘要
混合专家（MoA）架构通过将多个LLM智能体组织成多层推理流水线来改进推理时扩展。然而，现有的MoA变体在深度增加时无法维持增益，出现性能退化、早期平台期或饱和。我们提出ReM-MoA，一种记忆增强的MoA框架，通过两种机制维持扩展：（1）排名推理记忆，它使用比较评审智能体持续存储和排名来自所有层的推理轨迹；（2）策展多样化记忆路由方案，它将不同智能体暴露于成功和失败轨迹的不同组合中，在传播高质量推理的同时保持探索多样性。我们进一步引入可选的多领域评审者蒸馏流水线，通过前沿模型监督提高排名质量。在涵盖数学、形式逻辑、代码、知识和常识的五项推理基准测试中，ReM-MoA在深度和宽度扩展上均持续优于先前的MoA变体，且其优势随深度增加而扩大，确立了结构化的跨层推理记忆作为可扩展多智能体推理的关键缺失机制。

## Abstract
Mixture-of-Agents (MoA) architectures improve inference-time scaling by organizing multiple LLM agents into layered reasoning pipelines. However, existing MoA variants fail to sustain gains as depth increases, exhibiting degradation, early plateauing, or saturation. We propose ReM-MoA, a memory-augmented MoA framework that sustains scaling through two mechanisms: (1) a Ranked Reasoning Memory that persistently stores and ranks reasoning traces from all layers using a comparative Reviewer Agent, and (2) a Curated Diversified Memory Routing scheme that exposes different agents to distinct combinations of successful and failed traces, preserving exploration diversity while propagating high-quality reasoning. We further introduce an optional multi-domain Reviewer distillation pipeline that improves ranking quality through frontier-model supervision. Across five reasoning benchmarks spanning math, formal logic, code, knowledge, and commonsense, ReM-MoA consistently outperforms prior MoA variants across both depth and width scaling, and its advantage widens with depth, establishing structured cross-layer reasoning memory as a key missing mechanism for scalable multi-agent inference.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有的混合专家（Mixture-of-Agents, MoA）架构通过将多个LLM智能体组织成多层推理流水线来扩展推理时计算，但随着层数（深度）或每层智能体数（宽度）的增加，性能增益无法持续——出现三类失败模式：
  - **退化**：标准MoA在深度超过3层后准确率下降（如MMLU-redux上从61.5%降至52.4%）。
  - **早期平台期**：RMoA通过相邻层残差传播，避免退化但很快饱和，无法利用更早的高质量推理。
  - **饱和**：AttentionMoA通过层内智能体交互改善，但深层后性能停滞，且多样性崩溃。
- **根本原因**：现有方法未能同时保证**质量保持**（Q）和**多样性保持**（D）。原始拼接传播错误；局部通信丢失早期好轨迹；智能体输入趋同导致探索多样性丧失。
- **意义**：揭示结构化跨层推理记忆是可扩展多智能体推理的关键缺失机制。

## 2. 方法论：核心思想、关键技术、公式或算法流程

### 2.1 核心思想
- **ReM-MoA**：以“排名推理记忆”（Ranked Reasoning Memory）为中心，持久存储所有层的推理轨迹及其评分和理由，再通过“策展多样化记忆路由”（Curated Diversified Memory Routing）向不同智能体分发不同的高质量/低质量轨迹组合，**同时保持质量和多样性**。

### 2.2 关键技术细节

#### 排名推理记忆（Ranked Reasoning Memory）
- **存储器**：`M≤l` 记录了第1层到第l层的所有轨迹三元组（轨迹、分数、理由）。
- **评审智能体（Reviewer Agent）**：一个LLM，在每一层对所有N个提案者的轨迹进行比较评分（1-10分，S=10），并为每个轨迹生成针对性理由。采用比较评分而非独立评分以提升可靠性。
- **存储**：每层提交 `Ml = {(r_{l,j}, s_{l,j}, ρ_{l,j})}_{j=1}^N`。

#### 策展多样化记忆路由（Curated Diversified Memory Routing）
- **原理**：从记忆 `P_l = M≤l−1` 中为每个智能体 `A_{l,j}` 选择不同的参考集 `S_{l,j}`。
- **三大路由家族**：
  - **对比型（Con）**：`Top_a ∪ Bot_b`，组合最高分和最低分轨迹。
  - **正向强化（Top）**：仅取最高分轨迹。
  - **失败模式聚焦（Bot）**：仅取最低分轨迹。
- **实例**（N=3）：
  - 智能体1：`Con_2`（Top1 ∪ Bot1）
  - 智能体2：`Top_2`
  - 智能体3：`Bot_2`
- 对于更大N，按Table 5扩展各家族规模，保持每种路由类型均有覆盖。

#### 可选增强：评审者蒸馏（Reviewer Distillation）
- **数据生成**：在ReM-MoA自身运行中收集轨迹（L=3, N=3），用前沿教师模型（GPT-5.5）标注分数和理由。
- **训练**：对基础评审者（Qwen2.5-7B-Instruct）进行LoRA微调，最小化输出与教师标签的交叉熵损失。

#### 整体流程（文字描述）
1. 输入问题x给第1层提案者，生成N个轨迹 → 评审者评分并存入记忆。
2. 第2层起，每个提案者根据路由函数从记忆中选择参考集（含轨迹、分数、理由），结合问题x生成新轨迹 → 评审者再次评分并存入记忆。

## 6. 主要结论与发现

1. **持续扩展**：ReM-MoA在深度和宽度上均持续提升准确率，而所有基线在深度上要么退化（标准MoA）、要么平台期/饱和（RMoA, AttentionMoA）。优势随深度增加而扩大（如L=9时ReM-MoA比最强基线平均高4.92个百分点，比标准MoA高18.44个百分点）。
2. **质量-多样性联合保持**：排名记忆保证高质量传播，多样化路由保证探索差异，二者缺一不可。
3. **蒸馏收益**：多域评审蒸馏进一步提升性能（平均+1.92 @ L=9），且能迁移到训练未见的域（LOO恢复44%增益）。
4. **消融重要性排序**：跨层访问贡献最大（-4.28 @ L=9），路由多样化次之（-2.50 @ L=9），理由起细化作用（-1.52 @ L=9）。
5. **宽度的有限收益**：宽度扩展也能提升，但幅度小于深度，且所有方法均未在宽度上退化。

## 8. 不足与局限

- **宽度探索有限**：最大N=8，未检测更大宽度（如N=16）下是否出现饱和或退化。但匹配前人工作，计算约束可理解。
