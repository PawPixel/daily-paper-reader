## 摘要
大语言模型通过强化学习后训练获得了显著的推理能力。然而，现有的强化学习后训练通常依赖于均匀数据采样，这忽略了训练数据的语义结构和训练策略不断变化的能力。为了解决这些局限性，我们提出自适应数据调度（ADS），这是一个用于调整强化学习后训练节奏的双层数据调度框架，它以语义簇上的自适应分布和策略边界样本选择替代均匀采样。在簇层面，ADS根据语义模式组织样本，并维持自适应簇间分布以巩固当前训练进展。在样本层面，ADS执行簇内调度以持续采样策略边界样本，从而提供信息丰富的相对优势。在三个大语言模型和七个推理基准上的实验结果表明，相比分组相对策略优化（GRPO），ADS将平均准确率提升了5.2%。值得注意的是，ADS在不同目标设计的强化学习方法中均取得一致性改进，凸显其作为大语言模型强化学习后训练通用数据调度策略的潜力。源代码可在 https://github.com/Richard-zrx/ADS 获取。

## Abstract
Large Language Models (LLMs) achieve remarkable reasoning capabilities through reinforcement learning (RL) post-training. However, existing RL post-training commonly relies on uniform data sampling, which ignores the semantic structure of the training data and the changing capability of the training policy. To address these limitations, we propose Adaptive Data Scheduling (ADS), a dual-level data scheduling framework for pacing RL post-training that replaces uniform sampling with an adaptive distribution over semantic clusters and policy-boundary sample selection. At the cluster level, ADS organizes samples according to semantic patterns and maintains an adaptive inter-cluster distribution to solidify current training progress. At the sample level, ADS performs intra-cluster scheduling to continuously sample policy-boundary samples, which provides informative relative advantages. Experimental results across three LLMs and seven reasoning benchmarks demonstrate that ADS improves average accuracy by 5.2% over Group Relative Policy Optimization (GRPO). Notably, ADS consistently improves RL methods with different objective designs, highlighting its potential as a general data scheduling strategy for LLM RL post-training. The source code is available at: https://github.com/Richard-zrx/ADS.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究背景**：大语言模型（LLM）通过强化学习（RL）后训练（如 GRPO）显著提升了推理能力。然而，现有 RL 后训练普遍采用**均匀数据采样**，将整个训练集视为无差别的样本池。
- **核心问题**：均匀采样忽略了两个关键因素：
  - **语义结构**：不同样本属于不同语义类别（如几何、代数），策略在不同类别上的能力可能差异很大，均匀混合会导致策略在不同语义间反复切换，难以巩固某个类别的学习进展。
  - **策略能力的动态变化**：在 GRPO 框架中，只有**策略边界样本**（即当前能力刚好能部分解决的样本）才能产生有意义的相对优势（advantage）。过易或过难的样本会导致优势坍塌（全部正确或全部错误），提供无效梯度。
- **整体含义**：需要一种**自适应数据调度**机制，能够根据语义结构和策略当前能力动态安排训练样本，使学习始终处于**合适的节奏**上。

### 2. 论文提出的方法论：核心思想、关键技术细节

#### 核心思想
提出 **Adaptive Data Scheduling (ADS)**，一种**双层次自适应数据调度**框架：
- **簇级别（Cluster-level）**：将训练样本根据语义聚合为簇，动态调整各簇被选中的概率，优先训练当前策略**能力范围内的簇**以巩固进展。
- **样本级别（Sample-level）**：在每个选中的簇内部，进一步筛选**策略边界样本**（成功率为 50% 附近的样本），确保每步更新提供信息性梯度。

#### 关键技术细节

1. **语义聚类（Semantic Clustering）**
   - 使用**基座策略**（base policy）的最后一层隐藏状态对每个样本的 prompt+reference solution 进行编码，取均值池化得到嵌入向量。
   - 对嵌入向量执行 **K-Means 聚类**（K=64），将数据集划分为 K 个语义簇。

2. **簇间分布（Inter-Cluster Distribution）**
   - 维护每个簇的**在线成功率** \(r_k^m\)（当前步骤该簇所有 rollouts 的正确率）。
   - 簇的采样概率与成功率成正比：\(p_k^{m+1} \propto r_k^m\)，并加入**指数平滑**（\(\alpha=0.3\)）防止振荡。
   - 成功率高的簇（当前能力可驾驭）被优先选中；成功率低的簇（太困难）被暂时降权。

3. **簇内调度（Intra-Cluster Scheduling）**
   - 在每个簇内，预先计算每个样本的**离线难度**（基于基座策略对 reference solution 的负对数似然），并按难度排序。
   - 维护一个**边界迷你簇（boundary mini-cluster）**，初始时选取最易的 \(B=32\) 个样本。
   - 每次训练后，根据该迷你簇中每个样本的在线成功率 \(\rho\) 进行更新：
     - 若 \(\rho \in [0.33, 0.67]\)（策略边界），保留；
     - 若 \(\rho < 0.33\)（太难），替换为簇内**更难**的邻近样本；
     - 若 \(\rho > 0.67\)（太易），替换为簇内**更易**的邻近样本。
   - 每步实际训练时，仅使用采样到的簇的边界迷你簇中的样本（共 128 个样本，对应每步 batch size 128）。

4. **算法流程**（Algorithm 1）：
   - 预处理：计算嵌入、聚类、计算难度分数，初始化簇间分布和迷你簇。
   - 每训练步：按簇间分布采样簇 → 使用这些簇的迷你簇样本进行 GRPO 更新 → 根据 rollout 结果更新迷你簇和簇间分布。
   - 每个 epoch 开始时重置调度状态（重新初始化分布和迷你簇），防止历史统计主导。

1. **ADS 显著提升推理准确率**：在全部 3 个模型和 7 个基准上，ADS 的平均 Mean@16 相比 GRPO 提升 **5.2%**（从 21.57% 到 28.05%，Qwen2.5-1.5B 数据；其他模型类似）。

5. **训练动态**：ADS 呈现“螺旋式”reward 上升，局部波动但长期增长，且 entropy 更平滑，表明优化稳定。
