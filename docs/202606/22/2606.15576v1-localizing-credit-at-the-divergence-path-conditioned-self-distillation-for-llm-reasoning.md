## 摘要
基于可验证奖励的强化学习为每次探索分配单一标量，在长推理轨迹中留下未充分指定的token级信用分配。在线策略自蒸馏通过让同一模型作为教师，以特权信息为条件，产生密集的逐token信号来解决这一问题。但通常选择的地面真实答案只是一个终点线索：在简洁答案任务中，教师在中间位置保持沉默，而这些位置上的路径级指导最为重要。我们提出后见自蒸馏（Hindsight Self-Distillation, HSD），该教师基于当前训练组中抽取的成功同行探索结果作为条件。这样的同行是从成功条件策略中精确采样的样本，无需额外的采样探索。通过提供完整的成功延续而非仅最终答案，产生的信用信号集中在失败探索与成功同行之间的分歧位置。在Qwen3-8B和Qwen3-32B的数学和代码基准测试中，HSD在GRPO变体和在线策略蒸馏基线中取得了最佳结果，在AIME等简洁答案任务上提升最大。

## Abstract
Reinforcement learning from verifiable rewards assigns a single scalar to each rollout, leaving token-level credit assignment underspecified in long reasoning traces. On-policy self-distillation addresses this by letting the same model act as a teacher conditioned on privileged information, producing a dense per-token signal. But the common choice of a ground-truth answer is only an endpoint cue: on terse-answer tasks, the teacher falls silent at the intermediate positions where path-level guidance matters most. We propose Hindsight Self-Distillation (HSD), which conditions the teacher on a successful peer rollout drawn from the current training group. Such a peer is an exact sample from the success-conditioned policy, requiring no additional sampled rollouts. By providing a full successful continuation rather than only the final answer, the resulting credit signal concentrates at the divergence position between a failed rollout and a successful peer. Across Qwen3-8B and Qwen3-32B on math and code benchmarks, HSD obtains the best result against GRPO variants and on-policy distillation baselines, with the largest gains on terse-answer tasks such as AIME.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：基于可验证奖励的强化学习（RLVR）为每个完整轨迹分配单一标量奖励，但在长推理轨迹中，这一标量无法区分哪些 token 导致失败，导致 **token 级信用分配不明确**。
- **现有方法不足**：在线策略自蒸馏（OPSD、SDPO 等）通过让教师模型以 **特权上下文**（如地面真实答案 a*）为条件，产生逐 token 的密集信号。但在简短答案任务（如 AIME，答案仅 3 位数）中，答案只约束轨迹终点，中间步骤的 **教师预测几乎不发生变化**（即“教师沉默”），无法在需要路径级指导的位置提供有效梯度。
- **核心目标**：设计一种新的特权上下文，能够提供 **路径级信息**，使得信用信号集中在失败轨迹首次偏离成功路径的位置，从而提升长链推理中的信用分配效率。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想：后见自蒸馏（Hindsight Self-Distillation, HSD）

- **关键观察**：在组相对训练（GRPO）的每一轮中，组采样通常会同时产生 **失败轨迹** 和 **成功轨迹**。每个成功轨迹 o⁺ 是从当前策略的 **成功条件分布 π⁺** 中采样的新鲜样本，且无需额外采样成本。
- **方法**：对于当前组中的每个失败轨迹 oᵢ，HSD 将其教师条件设置为 **成功同行轨迹 o⁺ᵢ + 地面真实答案 a***（即 cᵢ = (a*, o⁺ᵢ)）。当组内没有成功同行时，降级为仅使用答案（cᵢ = a*）。
- **信用定位机制**：失败轨迹与成功同行共享前缀直到 **分歧位置 τ**。在 τ 之前，教师条件与共享前缀一致，信用信号接近零；在 τ 处，教师看到成功同行的下一步，而学生分布分散，产生尖锐的信用尖峰；在 τ 之后，轨迹已偏离演示，信用逐渐衰减。这使得梯度信号集中在最需要修正的位置。

### 关键技术细节

- **逐 token 信用公式**：
  \[
  A_{i,t} = \log \frac{\pi_\theta(o_{i,t} \mid q, c_i, o_{i,<t})}{\pi_\theta(o_{i,t} \mid q, o_{i,<t})}
  \]
  当 cᵢ 包含成功同行的连续路径时，该比率在分歧位置显著增大。
- **训练目标**：采用全词汇反向 KL 散度损失：
  \[
  \mathcal{L}_{i,t} = \sum_{v \in \mathcal{V}} \pi_{c_i,t}(v) \left[ \log \pi_{c_i,t}(v) - \log \pi_{i,t}(v) \right]
  \]
  加上对参考策略的 KL 惩罚项 β KL(π_θ ∥ π_0)。
- **算法流程**：每个训练步采样 G=8 个轨迹并评分；对于每个轨迹 i，若有成功同行则随机选一个（排除自身），将其与答案拼接作为教师输入；执行两次前向（学生和教师），一次反向更新。

## 8. 不足与局限

- **覆盖依赖**：只有当组内至少有一个成功同行时 HSD 才提供路径监督；在困难问题（成功率低）的早期训练阶段，降级为答案条件，这正是最需要路径指导的时候。论文未探索跨步缓冲过去成功以延长覆盖。
- **模型规模限制**：仅评估了 Qwen3-8B 和 32B，在更小或更大的模型上效果未知。
- **任务域限制**：仅针对数学推理和代码生成（可验证奖励），未涉及噪声奖励、多轮对话、指令微调起点等场景。
