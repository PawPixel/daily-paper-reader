## 摘要
技能记忆可通过将过往经验作为文本指导来改进智能体强化学习，但检索到的技能并非全知全能：它们可能在某个状态下有帮助，而在另一状态下却误导同一策略。这使得常见的特权教师假设变得脆弱，即技能条件提示可以被视为无技能提示的固定教师。我们提出UCOB，一种通过信用感知的在线双向自蒸馏来学习利用和演化智能体技能的框架。UCOB将技能条件提示和无技能提示视为同一模型的两个在线上下文视角，在相同任务和锚定状态下比较它们的剩余回报，并将更高回报的视角作为局部教师。这种局部信用信号将有用的技能条件行为内化，纠正误导性的技能使用，并引导任务/状态技能记忆更新、效用感知检索和反思自训练。在包括ALFWorld、WebShop和Search-QA等智能体任务上的实验表明，UCOB在不同模型规模下均优于无技能强化学习、技能记忆基线和自蒸馏方法，在ALFWorld和WebShop上相比最先进基线分别提升高达23.5和18.0个点。消融实验和分析进一步验证了其核心机制和效率。

## Abstract
Skill memories can improve agentic reinforcement learning by reusing past experience as textual guidance, but retrieved skills are not oracular: they may help in one state while misleading the same policy in another. This makes the common privileged-teacher assumption fragile, namely that a skill-conditioned prompt can be treated as a fixed teacher for the no-skill prompt. We introduce UCOB, a framework for learning to utilize and evolve agentic skills via credit-aware on-policy bidirectional self-distillation. UCOB treats skill-conditioned and no-skill prompts as two on-policy context views of the same model, compares their return-to-go within the same task and anchor state, and uses the higher-return view as the local teacher. This local credit signal internalizes useful skill-conditioned behavior, corrects misleading skill usage, and guides task/state skill memory updates, utility-aware retrieval, and reflection self-training. Experiments on agentic tasks, including ALFWorld, WebShop, and Search-QA, show that UCOB outperforms skill-free RL, skill-memory baselines, and self-distillation methods across model scales, with up to 23.5 and 18.0 point gains over SOTA baselines on ALFWorld and WebShop. Ablations and analyses further validate its core mechanisms and efficiency.

#### 1. 核心问题与整体含义（研究动机和背景）
- **研究动机**：在长时段交互的智能体任务中，重用过往经验（即技能记忆）能有效提升强化学习（RL）的性能。然而，现有方法（如SDAR、Skill-SD）普遍假定检索到的技能条件是“特权教师”，即技能条件提示可以固定地作为无技能提示的监督源。这一假设是脆弱的：技能可能在某些状态下有益，在另一些状态下却误导策略，且技能条件分支并不总是优于无技能分支。
- **核心问题**：如何在线判断技能在局部状态下的效用，并根据实际回报动态选择教师方向，从而既内化有用技能行为又纠正误导行为，同时演化技能记忆本身。
- **整体含义**：UCOB将技能利用重新定义为信用分配问题——智能体不仅需要学会检索什么样的技能，还需要学会何时信任、何时纠正、何时内化这些技能。

#### 2. 方法论
- **核心思想**：将技能条件提示和无技能提示视为同一在线策略的两个上下文视图，在相同任务和锚定状态（anchor state）下比较二者的剩余回报，选取更高回报的视图作为局部教师，进行双向自蒸馏（Credit-Aware Bidirectional Self-Distillation, CBSD）。
- **关键技术细节**：
  - **双粒度技能记忆与混合 rollout**：维护任务级和状态级技能池；使用UCB分数进行效用感知检索；每batch采集一半带技能、一半不带技能的轨迹。
  - **锚定状态分组**：按照相同任务和相同锚定状态将 rollout 记录分组，每个记录包含分支标识、提示、响应和剩余回报。
  - **CBSD**：在每个锚定状态组内，选择最高剩余回报的记录作为信用参考（winner），将其行为蒸馏到相反分支。对每个接受的对，采用 top-K 令牌支持匹配构造 KL 散度目标，并使用回报差值加权和置信门控。
  - **技能记忆演化**：利用CBSD的信用证据更新技能效用（EMA）；对反思写作器进行自训练（基于效用导出的优势进行策略梯度更新）。
  - **整体损失**：L = L_RL + λ_CBSD * L_CBSD + λ_refl * L_refl + β_KL * L_KL，结合在线RL、双向蒸馏、反思自训练和KL正则化。
- **算法流程（文字说明）**：
  1. 当前策略根据检索的技能生成技能条件提示和无技能提示；
  2. 混合采样两种提示下的轨迹，与环境交互得到奖励；
  3. 按任务和锚定状态分组，计算各记录剩余回报；
  4. 对每组选出最高回报记录作为教师，将教师的行为通过top-K KL散度蒸馏到相反视图（技能→无技能 或 无技能→技能）；
  5. 同时基于这些信用证据更新技能效用，并自训练反思写作器；
  6. 联合优化RL、蒸馏和反思损失。
