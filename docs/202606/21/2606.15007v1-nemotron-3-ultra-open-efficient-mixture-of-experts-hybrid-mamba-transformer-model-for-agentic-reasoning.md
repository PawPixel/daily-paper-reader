## 摘要
我们推出Nemotron 3 Ultra，一个总参数量5500亿、激活参数量55亿的混合专家混合Mamba-注意力语言模型。我们在20万亿文本token上预训练Nemotron 3 Ultra，然后将上下文长度扩展至100万token，并使用监督微调（SFT）、强化学习（RL）和多教师在线策略蒸馏（MOPD）进行后训练。Nemotron 3 Ultra是我们迄今为止能力最强的模型，采用了多项关键技术——LatentMoE、多token预测（MTP）、NVFP4预训练、多环境RLVR、MOPD以及推理预算控制。与最先进的开源大语言模型相比，Nemotron 3 Ultra在实现同等精度的同时，推理吞吐量提升高达约6倍。最先进的精度、高推理吞吐量和100万token上下文长度使Nemotron 3 Ultra成为长时间自主代理任务的理想选择。我们在HuggingFace上开源了基础模型、后训练模型和量化检查点，以及训练数据和配方。

## Abstract
We introduce Nemotron 3 Ultra, a 550 billion total and 55 billion active parameter Mixture-of-Experts Hybrid Mamba-Attention language model. We pre-trained Nemotron 3 Ultra on 20 trillion text tokens, then extended the context length to 1M tokens, and post-trained using Supervised Fine Tuning (SFT), Reinforcement Learning (RL), and Multi-teacher On-Policy Distillation (MOPD). Nemotron 3 Ultra is our most capable model yet, employing multiple key technologies - LatentMoE, Multi Token Prediction (MTP), NVFP4 pre-training, multi-environment RLVR, MOPD, and reasoning budget control. Nemotron 3 Ultra achieves up to ~6x higher inference throughput as compared to state-of-the-art publicly available LLMs while attaining on-par accuracy. The state-of-the-art accuracy, high inference throughput, and 1M token context length make Nemotron 3 Ultra ideal for long-running autonomous agentic tasks. We open-source the base, post-trained, and quantized checkpoints, along with the training data and recipe on HuggingFace.

*   **关键技术细节**：

        *   **混合Mamba-Transformer**：在Transformer层中插入高效的状态空间模型Mamba-2层。Mamba层负责处理大部分序列建模，避免了Transformer的自注意力机制带来的二次计算复杂度和KV缓存（KV Cache）膨胀问题。Transformer层作为稀疏的“注意力锚点”，提供全局信息。这种设计显著提升了长序列下的推理吞吐量。
        *   **LatentMoE**：一种改进的MoE路由机制，通过“潜在”表示进行专家路由，能以更低的计算/参数成本达到更高精度。
        *   **多Token预测（MTP）**：模型除了预测下一个Token外，还额外训练了多个预测头来同时预测后续的多个Token。这有两个好处：一是提升预训练效率，二是作为原生投机解码（Speculative Decoding）的草案模型，加速推理。

        *   **长上下文扩展**：在基础预训练后，使用持续预训练（CPT）将上下文长度扩展到**1,048,576（1M）**个Token。

    3.  **后训练（Post-training）**：
        *   **整体流程**：`基础模型 -> SFT -> RLVR -> MOPD Warmup -> MOPD -> MTP Boosting -> 最终模型`。
        *   **监督微调（SFT）**：使用涵盖代码、数学、科学、搜索、聊天、安全等多领域的混合数据进行两阶段SFT，建立基础能力。
        *   **强化学习与可验证奖励（RLVR）**：在统一的多环境（终端使用、软件工程、搜索、数学、代码、安全、聊天等）中进行RL训练，使用可编程或基于LLM判断的奖励信号。
        *   **多教师在策略蒸馏（MOPD）**：这是论文的核心创新之一。训练了**超过10个**不同领域的专家教师模型（如SWE教师、搜索教师、终端使用教师、STEM教师等）。然后，学生模型（Ultra）生成自己的轨迹，多名教师模型对这些学生生成的轨迹在Token级别上提供密集的指导信号（KL散度），将教师的专业知识蒸馏给学生。这个过程迭代进行，使学生模型能融合多个专家的能力，并提升泛化性。
        *   **推理效率与预算控制**：模型被训练了“普通推理”、“中等努力”等多个推理模式，允许用户在推理时动态调整计算量（Token数量）与准确性之间的权衡。

### 3. 实验设计

*   **使用的数据集**：
    *   **预训练数据**：拥有19个高级类别，包括质量过滤的网络爬虫数据（如FineWeb）、代码（包括新刷新的173B Tokens）、数学、学术文本、法律数据（多个合成数据集）、多语言数据等。论文开源了多个用于提升法律、代码等能力的特定数据集。
    *   **后训练数据**：在SFT、RL和MOPD阶段使用了大量合成和过滤数据，涵盖软件工程（SWE-bench, SWE-Gym等代码库）、科研搜索、终端使用、数学证明、对话、CUDA编程、RTL设计等领域。

### 6. 论文的主要结论与发现

1.  **架构有效**：MoE混合Mamba-Transformer架构能够同时实现**世界顶尖的准确率**和**极高的推理吞吐量**（相比其他SOTA开源模型高达**5.9倍**）。
2.  **后训练方法强大**：提出的多教师在策略蒸馏（MOPD）是一种有效融合多领域专家知识的方法，尤其在**智能体任务**上取得了显著提升，但在纯粹的知识推理任务上提升相对有限。
3.  **长上下文能力强**：模型成功支持**1M Token**的上下文，并在RULER等长上下文基准测试上表现优异。
4.  **全能型模型**：Nemotron 3 Ultra不仅擅长智能体任务，在各种推理、知识、代码、数学、多语言、指令遵循和长上下文基准上均表现出色，是能力全面的顶尖模型。
5.  **开源生态贡献**：开源了完整的模型权重（基础、后训练、量化版本）、训练数据（多项）、训练配方（Recipe）和强化学习环境，这对于推动整个社区的研究和发展具有巨大价值。

### 7. 优点

*   **高效率的混合架构**：将高效的Mamba-2层与强力的Transformer注意力层结合，是一个极具前瞻性和实用性的设计思路，有效破解了长序列推理的计算瓶颈。
*   **综合创新的后训练管线**：将SFT、RLVR与独创的MOPD技术相结合，实现了能力的全面强化和多领域知识的融合，而非简单的RL堆叠。
*   **严谨的实验设计与分析**：论文包含大量精心设计的消融实验，对每项技术选择都进行了量化验证和深入的原因分析（如MOPD在非Agentic任务上效果不佳的分析）。这对后续研究极具参考价值。
*   **全面的开源承诺**：不仅开放了模型权重，还开放了宝贵的训练数据和完整配方，极大促进了AI研究的可复现性和社区贡献。
*   **硬件与软件协同优化**：论文详细阐述了对GB200集群（NVLink拓扑、NUMA绑定、Ray启动等）的众多底层优化，展示了在超大规模训练系统中的深厚工程功底，确保了理论能在实践中落地。

### 8. 不足与局限

*   **MOPD的局限性**：论文坦诚地指出了MOPD的局限——**当教师模型的能力优势来自额外监督微调（SFT）数据，而非学生对轨迹的偏好时，MOPD效果不佳**。对于纯粹的推理任务，这种方法难以弥补信息鸿沟。
*   **训练不稳定性**：在超大规模预训练中仍然出现了两次训练发散，虽然都得到了分析和解决，但这也表明NVFP4预训练等技术在大规模应用时仍有稳定性挑战。论文未能完全解释第二次发散的根本原因。
*   **基础设施要求极高**：整个训练和部署流程都依赖于顶级的**GB200 NVL72集群**。文中详细描述的诸多基础设施上的“坑”和“微调”（如启动优化、JIT缓存等），对于不具备相同条件的团队来说是巨大的门槛。该工作的可复现性在非NVIDIA高端硬件上存疑。
*   **合成数据依赖**：后训练大量依赖由教师模型（如DeepSeek-V4, GPT-OSS, Qwen-3等）生成的合成数据。这引入了对第三方模型质量的依赖，并可能在知识边界或偏差上形成“继承”。
*   **学术理论深度有限**：虽然创新点很多，但大多数创新（如LatentMoE, MOPD, 混合架构）在科学社区已有先例（论文也引用了大量工作），这篇工作更像是将这些技术成功**规模化、工程化**的里程碑式记录，而非提出一个全新的科学理论。
*   **缺乏蒸馏教师成本讨论**：MOPD需要训练超过10个领域专家教师模型，这本身也需要巨大的计算和数据成本。论文没有讨论这部分“隐藏成本”，这可能使得MOPD的经济效益对于学术界或中小企业来说不那么有吸引力。
