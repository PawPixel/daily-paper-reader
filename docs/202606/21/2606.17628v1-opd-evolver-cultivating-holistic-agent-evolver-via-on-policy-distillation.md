## 摘要
记忆已成为自我进化智能体的标准基础，但保留经验与学习如何通过经验进化并非同一回事。现有记忆智能体能够存储轨迹、检索反思或积累技能，但往往缺乏选择有用经验、据此行动、编写可复用知识以及维护不断增长的知识库的整体能力。我们提出OPD-Evolver，一种通过在线策略自我蒸馏培养此类智能体进化器的快慢协同进化框架。在快循环中，OPD-Evolver与四级记忆层次结构交互，以读取、使用、写入和维护经验，实现快速的测试时进化。在慢循环中，结果校准的记忆归因和特权后见之明将这些能力蒸馏到可部署的策略中。在多领域基准测试中，OPD-Evolver超过ReasoningBank等记忆系统高达11.5%，超过Skill0等基于训练的方法约5.8%。进一步分析表明，OPD-Evolver内化了高价值经验和记忆管理，使OPD-Evolver-9B能够挑战Qwen3.5-397B-A17B和Step-3.5-Flash等庞大对手，指向超越记忆增强型智能体、迈向真正合格的智能体进化器。

## Abstract
Memory has become a standard substrate for self-evolving agents, yet retaining experience is not the same as learning how to evolve through it. Existing memory agents can store trajectories, retrieve reflections, or accumulate skills, but often lack the holistic competence to select useful experience, act on it, write reusable knowledge, and maintain a growing repository. We introduce OPD-Evolver, a slow-fast co-evolution framework that cultivates such an agent evolver through on-policy self-distillation. In the fast loop, OPD-Evolver interacts with a four-level memory hierarchy to read, use, write, and maintain experience for rapid test-time evolution. In the slow loop, outcome-calibrated memory attribution and privileged hindsight distill these four abilities into the deployable policy. Across multi-domain benchmarks, OPD-Evolver surpasses memory systems such as ReasoningBank by up to 11.5%, and training-based methods such as Skill0 by ~5.8%. Further analysis shows that OPD-Evolver internalizes high-value experience and memory management, enabling OPD-Evolver-9B to challenge giant counterparts such as Qwen3.5-397B-A17B and Step-3.5-Flash, pointing beyond memory-augmented agents toward genuinely qualified agent evolvers.

## 2. 方法论：核心思想、关键技术细节与算法流程

### 核心思想
采用**慢-快共进化**（slow-fast co-evolution）框架：
- **快循环**（Fast Loop）：在线交互中通过四级记忆层次完成读取、使用、写入和维护，实现测试时快速进化。
- **慢循环**（Slow Loop）：利用结果校准的记忆归因和特权后见之明，通过**在线策略自蒸馏**（on-policy self-distillation）将四种进化能力注入同一策略，使智能体在无特权反馈时也能自行进化。

### 关键技术细节

#### 1）四级记忆层次（Hierarchical Experience Memory）
记忆库由四种互补类型组成：
- `M_traj`（轨迹）：保留完整情节证据
- `M_tip`（提示）：存储局部警告或启发式规则
- `M_skill`（技能）：抽象可重用过程
- `M_tool`（工具）：存储可执行命令或代码模板

#### 2）快循环工作流程
- **检索与选择**：根据任务查询`zt=[xt; et]`，从每层检索Top-K候选（公式6），再通过选择器`Sel^ℓ_θ`压缩为上下文`ct`（公式7）。
- **执行**：策略`πθ`基于`ct`和交互历史生成动作`at,i`（公式8）。
- **写入**：同一策略根据任务、轨迹、奖励和所选记忆，决定向哪些层写入新记忆`Δ^ℓ_t`（公式9）。
- **维护**：每Q个任务触发一次存储库维护，执行查询、合并、删除操作（公式10）。

#### 3）结果校准归因（Outcome-calibrated Attribution）
利用任务反馈`Rt`估计每个记忆的价值`V(m)`（公式11-12）：
- 比较记忆被选中和未被选中时的平均回报差异
- 加入置信因子`γ(m)`和层级先验`α_ℓ(m)`
- 高值记忆成为“应被选择、使用、保留”的证据，低值记忆暴露应忽略的噪声

#### 4）统一后见自蒸馏（Unified Hindsight Self-Distillation）
对四种进化决策（选择、执行、写入、维护），学生仅见公开输入`zk`，教师额外见特权后见`hk`（公式13）。
- 学生采样在线输出`ŷk`（公式14）
- 计算学生与教师分布之间的token级KL散度`δ_{k,n}`（公式15）
- 目标函数为所有决策上的期望蒸馏损失（公式16）：`L_slow(θ) = Σ_k E[1/L_k Σ_n δ_{k,n}]`

### 算法流程（Algorithm 1）
1. **快循环**：每个任务`xt`：形成查询→检索选择→获得上下文→执行得到轨迹和奖励→写入记忆→每Q个任务执行维护，并记录日志。
2. **慢循环**：对每个已记录的记忆估计`V(m)`→为每种决策构建`(zk, hk)`→采样学生输出→最小化蒸馏损失→部署学生策略。

## 3. 实验设计

### 使用的数据集/场景
- **训练数据**（与评估基准不重叠）：
  - Agent World Model（AWM）：3,000个多轮工具使用环境
  - nvidia/Nemotron-Terminal-Corpus：2,000个终端风格交互
  - EnvScaler：2,000个可扩展工具交互环境

- **评估基准**（5个Benchmark，共10个子集）：
  - **LifelongAgentBench**：数据库(DB)和操作系统(OS)子集（各100任务）
  - **MemoryArena**：数学(Math)和物理(Physics)子集（354+86任务）
  - **AMA-Bench**：因果推理(CI)、状态更新(SU)、状态抽象(SA)子集（共208个情节，2,496个QA对）
  - **InterCode**：Bash（224）、CTF（100）、SQL（314）子集
  - **MiniHack**：Room、Maze、KeyRoom（各51任务）

### 对比方法
- **记忆增强方法（7种）**：ExpeL、AWM、Cheatsheet、MemP、ReasoningBank、EvolveR、MemEvolve
- **训练方法（5种）**：SFT、GRPO、Skill0、MemRL、Complementary RL
- **大型模型**：Qwen3.5-397B-A17B、Step-3.5-Flash（196B）

### 基线设置
所有记忆方法从空记忆库开始，仅从评估流中积累记忆；训练方法额外在MiniHack上用相同训练数据训练以确保公平。

## 6. 主要结论与发现

1. **整体性能领先**：OPD-Evolver在全部10个子集中超越同骨干的记忆基线，9B版本在OS（65% vs 61%）、AMA-SA（52.92% vs 48%）、InterCode-CTF（57% vs 53%）上提升显著。
2. **与大模型竞争**：OPD-Evolver-9B在9/10子集上超过Step-3.5-Flash（196B），在6/10子集上超过Qwen3.5-397B-A17B，表明通过进化能力培养，紧凑模型可以挑战巨模型。
3. **优于训练方法**：在MiniHack和InterCode的6个子集中，OPD-Evolver在5个上取得最佳，尤其在最难的Maze和KeyRoom上大幅领先GRPO和Complementary RL。
4. **蒸馏有效提升记忆质量**：选择蒸馏将选中记忆的中位数得分从0.66-0.69提升至0.76-0.79；写入蒸馏将写入记忆的中位数得分从0.80-0.82提升至0.89-0.91。
5. **经验内化成功**：去除外部记忆后，OPD-Evolver仍比原始骨干执行成功率更高、步数更少（提升3-7个点，减少最多2.5步），说明能力已内化到策略参数中。
