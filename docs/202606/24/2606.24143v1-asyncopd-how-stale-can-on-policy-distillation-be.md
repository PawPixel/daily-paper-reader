## 摘要
在策略蒸馏（OPD）利用教师反馈训练学生在其自身轨迹上进行学习，并在大语言模型（LLM）后训练中变得日益重要。然而，与强化学习（RL）类似，OPD面临在策略系统瓶颈，因为轨迹生成可能主导推理工作负载的训练时间。异步训练流程可通过将轨迹生成与学习器更新解耦来缓解此瓶颈，但这样做会引入陈旧策略数据。尽管先前研究已探索异步RL中的陈旧数据，但其在OPD中的影响仍未被充分研究。我们首次系统性地研究了异步OPD中的陈旧性问题，重点关注实际场景：教师反馈通过局部KL散度实现，且全词汇教师logits因存储或传输成本过高而不可行，因此需要有限教师分数缓存。我们首先表明，KL方向改变了陈旧数据问题：教师加权的正向KL对陈旧轨迹更鲁棒，而学生加权的反向KL则易受影响。其次，针对易受影响的逆向KL情况，我们研究了旨在稳定异步RL的方法能否缓解OPD陈旧性。在我们的实验中，它们并未优于更简单的OPD特定替代方案：在学习器时间下基于当前学生重新计算逆向KL信号。第三，我们分析了有限教师分数缓存如何为稀疏和采样的反向KL OPD估计器创建偏差-方差权衡。这促使我们使用多样本蒙特卡洛（MC）方法，它在保留MC可纠正性的同时减少单样本方差。最后，我们提出并开源AsyncOPD，一个基于这些估计器选择构建的完全异步OPD训练流程。实验表明，与严格同步训练相比，AsyncOPD将训练吞吐量提升了1.6倍至3.8倍，同时达到相当精度。

## Abstract
On-policy distillation (OPD) trains a student on its own rollouts guided by teacher feedback and is becoming increasingly important for large language model (LLM) post-training. Like reinforcement learning (RL), however, OPD faces an on-policy systems bottleneck, as rollouts can dominate training time for reasoning workloads. Asynchronous training pipelines can alleviate this bottleneck by decoupling rollout generation from learner updates, but doing so introduces stale-policy data. While prior work has studied stale data in asynchronous RL, its effects in OPD remain underexplored. We present the first systematic study of staleness in asynchronous OPD, focusing on a practical setting where teacher feedback is implemented through local KL losses and full-vocabulary teacher logits are too expensive to store or transfer, necessitating finite teacher-score caches. We first show that KL direction changes the stale-data problem: teacher-weighted forward KL is more robust to stale rollouts, whereas student-weighted reverse KL is vulnerable. Second, for this vulnerable reverse-KL case, we study whether methods designed to stabilize asynchronous RL can mitigate OPD staleness. In our experiments, they do not improve over a simpler OPD-specific surrogate: recomputing the reverse-KL signal under the current student at learner time. Third, we analyze how finite teacher-score caches create a bias-variance tradeoff for sparse and sampled reverse-KL OPD estimators. This motivates multi-sample Monte Carlo (MC), which preserves MC correctability while reducing one-sample variance. Finally, we present and open-source AsyncOPD, a fully asynchronous OPD training pipeline built from these estimator choices. Experiments show that AsyncOPD improves training throughput by $1.6\times$ to $3.8\times$ over strict synchronous training while reaching comparable accuracy.

## 1. 核心问题与整体含义（研究动机和背景）

**研究动机**：在策略蒸馏（On-Policy Distillation，OPD）是大语言模型（LLM）后训练中提升推理能力的重要方法，它使用教师模型对学生模型自身生成的轨迹提供密集的 token 级反馈。然而，OPD 面临与强化学习（RL）类似的**在策略系统瓶颈**：每个学习器更新必须等待当前模型生成的新轨迹。对于推理任务，这些轨迹长且昂贵，同步训练中生成阶段常成为瓶颈，导致学习器空闲。

**核心问题**：异步训练可以将轨迹生成、教师评分和学习器更新解耦并行，但引入了**陈旧策略数据**（stale-policy data）——学习器用较旧的学生模型生成的轨迹进行更新。虽然异步 RL 中研究了陈旧数据的处理方法，但 OPD 因教师反馈机制不同（需要局部 KL 散度、全词汇教师 logits 成本过高导致必须使用有限教师分数缓存），其陈旧性问题尚未被系统研究。

**整体含义**：论文首次系统性回答了异步 OPD 中三个关键问题：
- KL 方向的选择如何影响对陈旧轨迹的鲁棒性？
- 异步 RL 中稳定学习的替代目标能否迁移到 OPD？
- 有限教师分数缓存如何影响 OPD 估计器的设计（偏差-方差权衡）？

最终目标是构建一个高效、实用的异步 OPD 训练系统，在保持模型质量的同时大幅提升训练吞吐量。

## 2. 方法论：核心思想、关键技术细节

**核心思想**：通过分析异步 OPD 中陈旧性对正向 KL 和反向 KL 的不同影响，设计适用于反向 KL 的稳定策略梯度替代目标，并提出多样本蒙特卡洛（Multi-sample MC）估计器来缓解有限教师缓存带来的偏差-方差问题。最终将这些选择集成到完全异步的流水线 AsyncOPD 中。

### 关键技术细节：

#### (a) 正向 KL vs. 反向 KL 的陈旧鲁棒性分析
- **正向 KL**：`KL(q ∥ p_θ)`，权重由教师分布 q 决定。在异步场景中，教师分数缓存的是 q 的 top-k 或采样动作，正向 KL 只依赖于教师支持的动作，不依赖于陈旧学生采样的动作覆盖，因此对陈旧轨迹更鲁棒。
- **反向 KL**：`KL(p_θ ∥ q)`，权重由当前学生分布 p_θ 决定。当教师分数缓存来自陈旧学生时，当前学生支持的动作可能未在缓存中（支持不匹配，support mismatch），导致无法正确计算反向 KL 梯度，因此脆弱。

#### (b) 反向 KL 的策略梯度替代目标
论文对比了以下几种替代目标，发现**最优选择是“无裁剪+重算当前优势”的 OPD/IS 替代目标**：

| 方法 | 优势项来源 | 是否裁剪比率 | 说明 |
|------|-----------|-------------|------|
| `L_clip_old` | 使用缓存时的旧优势 `A_old` | 是 | PPO 风格直接迁移 |
| `L_noclip_old` | 使用旧优势 `A_old` | 否 | |
| `L_clip_θ` | 重算当前优势 `A_θ` | 是 | |
| `L_noclip_θ` | 重算当前优势 `A_θ` | 否 | **最佳** |
| Decoupled PPO [4] | 分离行为策略 | 是 | 异步 RL 方法 |
| M2PO [33] | 二阶矩约束 | 否 | 异步 RL 方法 |

**核心公式**：反向 KL 目标通过重要性采样可以写为：
`D_R(θ;s) = -E_{a~p_old}[ρ_θ(a,s) · sg(A_θ(a,s))]`，其中 `ρ_θ = p_θ(a|s)/p_old(a|s)`，`A_θ = log q(a|s) - log p_θ(a|s)`。该估计器无偏，但方差较大。论文实验表明，使用 `A_θ`（重算当前优势）且不裁剪比率，比使用缓存旧优势 `A_old` 或加入裁剪更好，也比异步 RL 的专用替代目标更优。

#### (c) 有限教师缓存下的估计器设计
- **稀疏 top-k**：缓存陈旧学生 top-k 动作的教师分数。由于陈旧学生的 top-k 支持可能与当前学生不同，重新加权无法弥补缺失的动作，存在固定支持不匹配偏差。
- **单样本蒙特卡洛（MC1）**：从陈旧学生分布采样一个动作并缓存教师分数。通过 IS 可以无偏估计当前反向 KL 梯度，但方差高。
- **多样本蒙特卡洛（Multi-sample MC）**：在每个解码步中，从陈旧学生分布采样 m 个动作（不展开完整轨迹），缓存它们的教师分数和旧 logits。学习器平均 m 个 IS 梯度，保留无偏性同时降低方差。实验显示 m=4 即可明显改善。

#### (d) AsyncOPD 流水线
采用 **AReaL 式完全异步架构**：工人持续生成轨迹，教师异步评分，学习器在批次就绪后立即更新，通过队列深度参数 τ 控制流水线中未消费的最大批次数量。权重同步时，进行中的请求保留前缀并重置缓存，而非丢弃。不同于 step-off 调度（固定 k 步延迟且等待整批完成），AsyncOPD 流式传输样本，消除长尾等待。

## 6. 主要结论与发现

1. **KL 方向决定陈旧鲁棒性**：正向 KL 因教师加权对陈旧轨迹鲁棒，反向 KL 因学生加权易受影响。反向 KL 精度随陈旧度增加下降更快。

2. **最优反向 KL 替代目标**：最有效的方法是**重算当前优势 `A_θ` 且不使用裁剪**。PPO 风格裁剪仅对使用旧优势 `A_old` 有帮助，但对 `A_θ` 有害（裁剪有用信号）。异步 RL 方法（Decoupled PPO、M2PO）并未提升性能。

3. **有限教师缓存下 MC 优于稀疏 top-k**：陈旧 top-k 存在支持不匹配偏差，即使重新加权也无法纠正。单样本 MC 通过 IS 无偏但高方差；**多样本 MC 降低方差同时保留无偏性**，m=4 即可显著提升。

4. **AsyncOPD 提升吞吐量 1.6×–3.8×**，同时保持与严格同步相当的精度（在多数设置中达到最佳或次佳水平）。流水线重叠度从约 0.8 提升至 2.2–2.6（最大 3.0）。

## 7. 优点

- **首次系统性研究异步 OPD 陈旧性**，填补了现有工作空白。
- **分析方法清晰**：将陈旧性分解为策略梯度替代目标轴和缓存支持轴，分别实验，归因明确。
- **估计器设计扎实**：从理论 IS 无偏性出发，提出多样本 MC，并实验验证方差降低；对比稀疏 top-k 的偏差本质。
- **实验广泛且公平**：覆盖多种陈旧度、多种替代目标、多种学生规模，使用严格受控的缓存深度协议。
- **提出实用的 AsyncOPD 流水线**并开源，证明了实际可部署的吞吐量提升。
- **识别出简单有效的缓解方法**（重算 `A_θ` + 多样本 MC），优于复杂的异步 RL 方法，便于直接应用。

## 8. 不足与局限

- **未覆盖全词汇密集 KL**：由于通信和存储成本，论文专注于稀疏 top-k 和 MC 估计器。全词汇 KL 在异步设置下如何处理陈旧性仍为开放问题。
- **单节点实验**：所有实验限于 8 块 B200 的单节点，未扩展到多节点集群。流水线的可扩展性、跨节点通信开销等未验证。
- **仅评估数学推理任务**：使用 DeepMath 数据集和 AIME/AMC 基准，未覆盖通用领域（如对话、代码生成、摘要等），结论的泛化性需进一步验证。
- **教师模型固定且仅单一大小**：仅使用 Qwen3-30B-A3B-Instruct-2507 作为教师；不同教师能力、架构、KL 特征可能改变结果。
- **未探索更激进的陈旧度**：最陈旧到 128 batch 步（约 512 次优化更新），更高度陈旧（例如数百 batch 步）的效果未知。
- **缺乏长期稳定性分析**：异步训练可能引入累积偏差或不稳定性，论文只训练了 200 次迭代，未研究更长训练时间下的收敛行为。
- **未与同步基线在相同 GPU 时间下公平比较**：同步训练在给定相同壁钟时间内更新次数更少，论文主要对比固定训练步数下的精度和吞吐量，但异步的更快达到相同精度已是优势。
