## 摘要
对于多步交互环境中的LLM智能体，一个关键挑战是有效利用累积的交互经验。现有工作通常将这种经验分为两种用途：将其作为自然语言规则保留在模型外部以便后续提示，或者使用轨迹和反馈来更新模型参数。前者易于解释，但可能无法与不断演化的策略保持同步；后者更广泛地改进策略，但在稀疏奖励设置中仅对局部错误提供有限的纠正。我们提出了基于经验规则与策略的联合学习用于LLM智能体（JERP），该方法从相同的交互轨迹中更新长期经验规则池和策略。在决策时，JERP检索任务相关规则，并结合交互历史对智能体进行条件化。每个回合后，它使用收集的轨迹来优化策略，并通过将当前 rollout 与参考成功轨迹进行比较来修订规则池。这种耦合使规则池与演化中的策略保持一致，同时允许稳定有效的行为逐渐被吸收到模型本身中。在AlfWorld和WebShop上的实验表明，JERP在复杂交互任务的决策性能上取得了一致的提升。

## Abstract
For LLM agents in multi-step interactive environments, a key challenge is to make effective use of accumulated interaction experience. Existing work has typically separated two uses of such experience: keeping it outside the model as natural-language rules for later prompting, or using trajectories and feedback to update the model parameters. The former is easy to interpret but can fall out of sync with the evolving policy; the latter improves the policy more broadly but provides only limited correction for local mistakes in sparse-reward settings. We present Joint Learning of Experiential Rules and Policies for LLM Agents (JERP), which updates a long-term experiential-rule pool and the policy from the same interaction trajectories. At decision time, JERP retrieves task-relevant rules and conditions the agent on them together with the interaction history. After each episode, it uses the collected trajectories both to optimize the policy and to revise the rule pool by comparing current rollouts with reference successful trajectories. This coupling keeps the rule pool aligned with the evolving policy while allowing stable and effective behaviors to be gradually absorbed into the model itself. Experiments on AlfWorld and WebShop show that JERP yields consistent gains in decision performance for complex interactive tasks.

- **核心问题**：大型语言模型（LLM）智能体在多步交互任务中如何有效利用累积的交互经验？现有方法将经验分为两种用途：一是作为外部自然语言规则存储，供后续提示使用（如反思、规则库）；二是通过轨迹和反馈直接更新模型参数（如强化学习微调）。这两种方式存在互补的局限：
  - 外部规则易于解释，但可能随策略演化而变得过时或失配；
  - 参数更新可全局改善策略，但在稀疏奖励下对局部错误修正不足。
- **整体含义**：论文提出**联合学习**策略，让**同一批交互轨迹同时用于规则池更新和策略优化**，使规则池与演化策略保持对齐，并将稳定有效的行为逐步吸收到模型参数中，从而兼顾可解释性与全局优化。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- **JERP（Joint Learning of Experiential Rules and Policies）**：维护一个**长期经验规则池**，每回合从规则池中选取**工作规则集**，与交互历史一起作为决策上下文。回合结束后，采样得到的轨迹组**同时**用于：
  1. **策略参数更新**（基于组相对优化，GRPO风格）；
  2. **经验规则池更新**（通过对比当前轨迹与参考成功轨迹，生成规则编辑操作）。

### 2.2 关键技术细节
- **规则池组织**：对每个任务实例d，规则池K(d)包含若干规则文本及其效用分数。每回合选择**效用分数最高的k条**作为工作规则集eK(d)。
- **轨迹采样**：在固定工作规则集下，策略πθ与环境交互生成N条完整轨迹。
- **策略参数更新**（阶段I）：
  - 对轨迹组内各轨迹的最终奖励标准化，得到组相对优势Ai = (R(τi) - μd) / (σd + δ)。
  - 所有时间步共享同一轨迹级优势。
  - 优化目标为GRPO损失（包括裁剪的代理项和KL散度正则化），使用LoRA进行参数高效微调。
- **规则池更新**（阶段II）：
  - 基于当前规则池、当前轨迹组及参考成功轨迹（累计的已成功轨迹），利用**冻结参数的LLM**生成结构化规则编辑操作，包括：
    - **ADD**（添加新规则）、**EDIT**（修改现有规则）、**UPVOTE** / **DOWNVOTE**（调整效用分数）、**MERGE**（合并语义相似规则）。
  - 根据操作应用修改，更新规则分数；低于阈值则移除；新规则赋予初始正分数；合并规则取原最大分数。
  - 规则池容量受限，需修剪。

### 2.3 算法流程概述（Algorithm 1）
- 初始化每个任务的规则池和成功轨迹集为空。
- 对每个训练步：
  - 采样任务实例d，构建工作规则集。
  - 采样N条轨迹，计算轨迹奖励。
  - 计算组相对优势，用GRPO损失更新θ。
  - 调用`ReflectAndEdit`生成编辑操作，更新规则池。
  - 更新参考成功轨迹集（加入当前回合成功轨迹）。

## 7. 优点

- **方法创新**：首次在LLM智能体中联合优化规则池和策略参数，实现两种经验利用方式的互补。
- **可解释性**：规则以自然语言形式存在，可观测、可编辑、可审计。
- **效果好**：在多个复杂交互任务上一致优于现有方法，尤其对稀疏奖励、长序列任务增益显著。

- **稳定性设计**：引入效用分数、合并操作、剪枝策略，防止规则池无序膨胀和过时。
